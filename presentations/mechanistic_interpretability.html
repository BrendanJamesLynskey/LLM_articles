CONTEXT

The Black Box Problem

A trained LLM is billions of numbers learned by gradient descent. The weights produce coherent text, reasoning, code — but the relationship between specific weights and specific behaviors is opaque.

What Other Methods Do

  • Behavioral: ask & analyze answers
  • Attribution: gradient saliency maps
  • Probing: train classifiers on hidden states
  • None reveal the actual algorithm

Mechanistic Goal

  • Identify the actual computation
  • Which neurons fire under which conditions
  • What role each plays in the larger algorithm
  • Circuit diagrams, not heatmaps

INDUCTION

Induction Heads

The First Big Find (2022)

  • Pairs of attention heads in transformers
  • Implement: copy patterns from earlier in the input
  • Mechanism cleanly identifiable in specific heads
  • Verified by ablation

How They Work

  1. Previous-token head: shifts attention to previous token
  2. Induction head: looks for earlier match of [previous token], copies what came after it
  3. Net effect: completes patterns A B ... A → B

Why It Mattered

Demonstrated mechanistic interpretability could produce real, verifiable explanations of model behavior. Skeptics’ “too complex to understand” was partially refuted.

SAE

Sparse Autoencoders

The Setup

  • Pick a layer of the model
  • Take activations across millions of inputs
  • Train SAE: encode to sparse vector, decode back
  • Hidden layer is over-complete with sparsity penalty

The Hope

Sparse hidden dimensions correspond to interpretable features: concepts like “France,” “numerical context,” “deception.”

The Theory

  • Superposition hypothesis: models represent more concepts than they have neurons by overlapping representations
  • SAE is a learned decoder that disentangles superposed features
  • Compressed sensing theoretical basis

MONO

Scaling Monosemanticity

Anthropic Papers

  • “Towards Monosemanticity” (Oct 2023) — small model
  • “Scaling Monosemanticity” (May 2024) — Claude 3 Sonnet
  • Millions of identifiable features
  • Surprising specificity

Memorable Features Found

  • Golden Gate Bridge feature
  • Code vulnerability features
  • Emotional tone features
  • Specific historical figures
  • Deception-related features

Golden Gate Claude

By clamping the Golden Gate Bridge feature high, researchers produced a model that wove the bridge into every response. Direct, mechanistic intervention on a specific concept — viral demo, real proof of concept.

CIRCUITS

From Features to Circuits

The Next Step

  • Features identify what
  • Circuits explain how
  • Groups of features + weights working together
  • Implements specific computations

Toy Model Findings

  • Modular arithmetic algorithms
  • Sorting circuits
  • Pattern matching circuits
  • Often surprisingly elegant

Frontier-Scale Difficulty

  • Combinatorial blowup of feature interactions
  • Full circuit identification infeasible
  • Compromise: narrow-task circuits
  • How does the model predict open-paren?
  • How does it complete arithmetic?

FINDINGS

What We’ve Learned

Surprises

  • Models represent abstract concepts, not just surface features
  • Computation is highly distributed
  • Models contain features for things they shouldn’t
  • Some computations are elegant; others are wasteful

Implications

  • Genuine semantic representations form during training
  • Suppressed concepts are still latent
  • Behavior can be inspected and intervened upon
  • Detection of misalignment becomes plausible

LIMITS

The Skepticism

Real Limits

  • Scale — harder at frontier model size
  • Completeness — spotlights, not maps
  • Frontier coverage incomplete
  • Some features may be artifacts of the SAE method

The Defense

  • Intervention experiments work
  • Behavior predictable from features
  • Robust across SAE training runs
  • Empirical evidence supports realness

Practical Applications

Emerging Uses

  • Model debugging (fine-tuning regressions)
  • Safety verification (monitor unsafe features)
  • Targeted interventions (suppress biases)
  • Knowledge editing (modify circuits storing facts)

The Anthropic Investment

  • Treats interpretability as core safety research
  • Tools: Garcon, Neuronpedia
  • Frontier-scale SAE deployments
  • Strategic priority, not side project

SUMMARY

The Bet on Understanding

The Bigger Picture

  • Other safety approaches (RLHF, constitutional AI, filters) work without understanding
  • Mechanistic interpretability bets that understanding is achievable
  • If it works, provides guarantees black-box approaches cannot
  • If it doesn’t, we deploy ever-more-powerful systems whose internals stay mysterious

Practitioner’s Lesson

Not yet plug-and-play. Worth following if you build verifiable or high-stakes systems. Early adopters of usable interpretability tools will have advantages. For most application developers, mechanistic interpretability remains a research topic — but its outcomes will shape the broader trustworthiness of AI systems everyone is building on top of.
