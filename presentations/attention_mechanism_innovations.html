The Core Problem

Standard self-attention computes scores between all N² token pairs, creating a quadratic bottleneck in both compute and memory.

4K Context

16M attention entries per head

128K Context

16B attention entries per head

KV Cache (70B)

~5 GB for a single 4K sequence with MHA

Every innovation in this report addresses one or both of these pressures: quadratic compute cost and KV cache memory.

Multi-Head Attention (MHA)

The original design runs h parallel attention heads, each with independent Q, K, V projections into dimension dk = dmodel / h.

Attention(Q, K, V) = softmax(QKᵀ / √d_k) V
  • Each head learns different patterns: syntactic, semantic, positional
  • Outputs concatenated and projected to dmodel
  • KV cache: h × d_k × 2 values per token per layer

MQA and Grouped-Query Attention

Multi-Query Attention (MQA)

  • Single KV head shared across all query heads
  • KV cache reduced by h× (e.g. 32× for 32-head model)
  • 0.5–1.0% perplexity degradation vs MHA
  • Adopted: PaLM, Falcon, StarCoder

Grouped-Query Attention (GQA)

  • Query heads grouped into g groups, each sharing one KV head
  • Sweet spot: 4:1 to 8:1 query-to-KV ratio
  • Minimal quality loss with large cache savings
  • Adopted: Llama 2/3, Mistral, Gemma 2, Qwen 2

Uptraining: MHA to GQA

Existing MHA models can be converted to GQA by mean-pooling adjacent KV heads within each group, then fine-tuning for only 5–10% of original training compute. This made GQA adoption practical without training from scratch.

Multi-Latent Attention (MLA)

DeepSeek-V2/V3 compresses the joint KV representation into a low-rank latent vector instead of sharing heads.

  • Input projected to compressed latent: c_t = WDKV · x_t
  • Full K, V reconstructed via up-projection during attention
  • 128 heads, dk=128: KV cache 64× reduction (32,768 → 512 values)
  • Virtually no quality loss vs full MHA
  • RoPE absorbed into query side — latent is position-independent

Efficient Attention Patterns

Sliding Window

  • Each token attends to w most recent tokens only
  • O(N × w) complexity — linear in N
  • Mistral 7B: w = 4,096 with full attention at periodic layers
  • KV cache bounded regardless of sequence length

Sparse Attention

  • Fixed patterns: strided, block-sparse (Sparse Transformer)
  • Learned patterns: routing via query-key clustering
  • BigBird: local + global + random = Turing complete
  • NSA (2025): hardware-aligned learned block selection

Dilated Attention

Attend to w tokens at intervals of dilation factor d. LongNet uses exponentially increasing dilation across heads for multi-scale coverage at O(N) cost.

Linear Attention

Performer, cosFormer: kernel approximations avoid N×N matrix. O(Nd²) complexity. Limited adoption due to 1–3 PPL quality gap on retrieval tasks.

Positional Encoding: RoPE Dominance

Rotary Position Embeddings (Su et al., 2021) apply position-dependent rotations to Q and K vectors.

(R_m q)^T (R_n k) = q^T R_{n-m} k
  • Dot product depends only on relative position m−n
  • Compatible with FlashAttention and efficient kernels
  • Rotation frequencies tunable for context extension

Adopted by: Llama 1/2/3, Mistral, Qwen, Gemma, Phi, Yi, DeepSeek, InternLM

Context Extension

  • PI: scale indices down by extension factor
  • NTK-aware: non-uniform scaling preserving high frequencies
  • YaRN: NTK + temperature adjustment — most widely used

ALiBi (Legacy)

Linear distance bias on scores. Good extrapolation but rigid. Superseded by RoPE + interpolation. Used by BLOOM, MPT.

FlashAttention: IO-Aware Computation

Restructures attention to work on tiles fitting in GPU SRAM rather than materializing the full N×N matrix in HBM.

  • Online softmax trick: exact attention computed incrementally
  • O(N) memory with zero approximation
  • Universal default in all major frameworks

Evolution

  • FA-1: IO-aware tiling
  • FA-2: better parallelism, 50–73% A100 peak
  • FA-3: H100 Hopper features, 1.2+ PFLOPS

Ring Attention

Distributes attention across devices in a logical ring. Overlaps communication with compute. Enables 1M+ token contexts across GPU clusters.

PagedAttention (vLLM)

Virtual memory for KV cache. Fixed-size pages allocated on demand. Eliminates 60–80% memory waste from fragmentation. Copy-on-write for shared prefixes.

Softmax Alternatives & Gating

Softcapping

  • Used in Gemma 2
  • tanh-based clamp on attention logits
  • Prevents extreme values before softmax
  • Improves training stability at scale

Log-n Attention

  • Scales logits by ratio of current to reference sequence length
  • Compensates for entropy growth in longer contexts
  • Better length generalization

Differential Attention

  • Two softmax heads subtracted from each other
  • Cancels noise, surfaces signal
  • Reduces hallucination
  • Improves long-context extraction

These modifications are individually small but meaningfully improve stability and quality, particularly for long-context training and inference.

SwiGLU & RMSNorm

SwiGLU Feed-Forward

Replaces the original ReLU FFN between attention layers.

  • Element-wise product of Swish-activated projection and linear gate
  • ~50% more parameters than standard two-layer FFN at same hidden dim
  • Smoother gradients, consistently better quality
  • Interaction with GQA/MLA defines the parameter allocation of each block

RMSNorm

Replaces LayerNorm in most modern transformers.

  • Normalizes by root mean square without mean subtraction
  • ~50% less computation than LayerNorm
  • Equivalent training stability
  • Combined with pre-normalization (norm before attention, not after)
  • Used by Llama, Mistral, Gemma, and most recent families

Together, SwiGLU + RMSNorm + GQA/MLA define the modern transformer block recipe adopted by nearly all leading model families.

Alternative Architectures

Mamba (SSM)

  • Selective state spaces: input-dependent transitions
  • O(N) training, O(1) per-step inference, no KV cache
  • Competitive quality up to 2.8B params
  • Mamba-2: connects SSMs and attention theoretically

RWKV

  • RNN-transformer hybrid with linear complexity
  • WKV mechanism: exponential decay favoring recency
  • Trained up to 14B parameters
  • Weak on needle-in-a-haystack retrieval

Jamba (Hybrid)

  • Interleaves attention (GQA) + Mamba layers
  • 1 attention layer per 3–4 Mamba layers
  • 8× KV cache reduction vs pure attention

RetNet & xLSTM

  • RetNet: exponential decay, 3 computation modes
  • xLSTM: exponential gating + matrix memory
  • Both approach transformer quality at moderate scales

Composability Across the Stack

Modern LLM architectures compose multiple innovations at different levels:

Architecture Level

GQA or MLA determines head structure and KV cache footprint

Position Level

RoPE + YaRN provides robust context extension

Implementation Level

FlashAttention eliminates attention matrix memory overhead

Serving Level

PagedAttention manages KV cache allocation; KV quantization (INT8/FP8) composes with any head design

Sparse patterns interact with FlashAttention via block-sparse masking — skipped tiles yield both compute and IO savings.

Inference System Implications

Bandwidth Bottleneck Shift

  • MHA decode: bottleneck is KV cache bandwidth
  • MQA / aggressive GQA: KV cache shrinks, bottleneck shifts to weight memory bandwidth
  • Performance-limiting factor becomes streaming model weights through compute units

Practical Consequences

  • MQA/GQA systems sustain larger batch sizes before exhausting memory
  • Optimal hardware config changes: more bandwidth vs more memory
  • KV cache quantization (FP8/INT8) amplifies these effects
  • Sliding window essential for streaming / unbounded contexts

Deployment Scenario Guide

  • High throughput (many users): prioritize KV cache efficiency — GQA, MLA, quantized caches
  • Very long context (100K+): sparse or hybrid attention essential
  • Low latency (single request): FlashAttention on dense attention may suffice

Emerging Research Frontiers

Native Sparse Attention

Learned block-level token selection with hardware-aligned gating. Sub-quadratic complexity, minimal quality loss.

Mixture of Attention Heads

Route tokens to specialized heads (local, global, positional). MoE principles applied to attention for dynamic compute allocation.

Attention Sinks

First tokens accumulate disproportionate weight. StreamingLLM: keep sink + window for constant-memory infinite generation.

Adaptive Computation

Early exit, layer skipping, dynamic token pruning. 20–50% compute reduction on typical workloads.

Attention-Recurrence Convergence

Mamba-2 = structured masked attention. The attention/RNN dichotomy may dissolve into a unified framework of sequence mixing operators.

Hardware Trends & Future

  • GPU SRAM growth (20 MB A100 → 50+ MB H100) improves FlashAttention tile sizes
  • Memory bandwidth grows slower than compute — GQA, MLA, KV quantization become more valuable
  • FP8 and FP4 tensor cores enable aggressive attention quantization
  • Compiler-driven kernel generation (Triton, Pallas) democratizes custom attention variants

Hardware Co-Design

Future accelerators may include dedicated softmax units, larger SRAM, and native support for online accumulation patterns.

Kernel Fusion

Combine attention + LayerNorm + residual + RoPE + FFN into single kernels. Reduces HBM round-trips and launch overhead.

The Open Question

Attention's ability to perform in-context learning appears deeply connected to its softmax-weighted retrieval structure. Understanding which components are essential will guide the next generation of architectures.

Key Takeaways

Head Design

GQA (4:1–8:1) is the de facto standard. MLA offers superior compression with no quality loss. Uptraining enables MHA→GQA conversion at 5–10% compute cost.

Positions & Stability

RoPE + YaRN dominates positional encoding. Softcapping, log-n attention, RMSNorm, and SwiGLU form the modern block recipe.

Computation

FlashAttention is universal. PagedAttention eliminates memory waste. Ring attention enables million-token contexts.

Alternatives

Hybrid architectures (Jamba) blend attention + SSMs. The attention/recurrence boundary is blurring. Practitioners compose techniques across the full stack.

As models and contexts grow, attention mechanism innovation remains at the center of LLM architecture research.
