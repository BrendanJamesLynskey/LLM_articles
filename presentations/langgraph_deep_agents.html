deep-agents / 01
PROBLEM

The Long-Horizon Problem

Why ReAct alone breaks

  • Context bloat — old tool outputs consume tokens needed for reasoning
  • No persistent plan — plan drifts out of the message history
  • No separation — one model plans, executes, summarises, writes, mediocre at all
  • Works fine for 5 tool calls; collapses at 50

What strong long-horizon agents do

  • Explicit plan — a todo list the agent maintains
  • Virtual filesystem — durable scratchpad outside chat
  • Sub-agents — isolated context per sub-task
  • Detailed system prompt that teaches when to use the above

Pattern seen in Claude Code, Deep Research, Manus, agentic coding benchmarks.

deep-agents / 02
FOUR PILLARS

The deepagents Formula

1. Todo list

write_todos(todos: list)

Explicit plan the model writes at start, revises, marks in_progress / completed.

2. Virtual FS

ls / read_file / write_file / edit_file

In-state dict: path → content. Sandboxed & persisted in state.

3. Sub-agents

task.<name>(...)

Scoped, fresh-context sub-agent as a tool. Shared FS + todos; distinct messages.

4. System prompt

~500 tokens teaching the model when to use each of the above. Essential — without it the tools are ignored.

deep-agents / 03
API

One-Call Constructor

from deepagents import create_deep_agent

agent = create_deep_agent(
    tools=[search_web, fetch_url],
    instructions="You are a research analyst...",
    subagents=[
        {
            "name": "research",
            "description": "Research a specific topic and return a summary with citations.",
            "prompt": "You are a research agent. Search, read, return a concise sourced summary.",
            "tools": ["search_web", "fetch_url"],
        },
        {
            "name": "critic",
            "description": "Review a draft for accuracy.",
            "prompt": "You are a critic. Read the draft, return concrete suggestions.",
            "tools": ["read_file"],
        },
    ],
    model=...,                 # defaults to Claude Sonnet 4.5
    builtin_tools=["write_todos", "ls", "read_file", "write_file", "edit_file"],
).compile(checkpointer=checkpointer)

Returns a normal LangGraph graph — you get checkpoints, HITL, streaming, tracing for free.

deep-agents / 04
STATE

Extended State

DeepAgentState

class DeepAgentState(AgentState):
    files: Annotated[dict[str, str], file_reducer]
    todos: Annotated[list[Todo], todo_reducer]

Both fields persist via the normal checkpointer. A resumed thread keeps every file and the in-progress plan.

Anatomy of a run

  1. write_todos drafts the plan
  2. Search + fetch tools gather material
  3. write_file("/notes/x.md", ...) saves findings
  4. Subtopic → task.research(...) sub-agent
  5. read_file + write_file("/report.md", ...) synthesises final answer
  6. Todos marked completed

OPS

Failure Modes & Mitigations

Failure modeSymptomMitigation
Plan driftModel ignores its own todo list after a few turnsStrengthen system prompt; surface plan in UI
Filesystem thrashSame file rewritten repeatedly, not convergingWrite-counter in state; diagnostic node on threshold
Sub-agent context lossSub-agent asked to do something parent forgot to passParent passes all relevant context in invocation arg
Runaway costsLarge sub-agent fan-out on cheap-planning/expensive-executionBudget tracking in state; middleware budget check
Weak modelTools called but plan never draftedUse Sonnet-class or better; cheaper models for sub-agents only

All four failure modes are visible in LangSmith traces. Enable tracing from day one.

deep-agents / 08
COST

Performance Tradeoffs

Overhead

  • System prompt: +500-800 tokens per model call
  • Extra FS & todo tools in tool list
  • Sub-agent spawn = one extra model call + its own context
  • FS round-trips each require a tool call

Prompt caching is essentially mandatory — makes the system prompt effectively free after the first call.

When it pays off

  • Task would fail in a plain ReAct loop
  • 50+ tool calls required
  • Multiple independent subtopics
  • Durable deliverable (report, codebase, analysis)

For short tool-use loops: use prebuilt ReAct. Deep Agents is for long horizons.

deep-agents / 09
TAKEAWAYS

Summary

Deep Agents in one line

A LangGraph graph wrapping one capable model + (todo + virtual FS + sub-agent) tools + a system prompt that makes the model actually use them.

Use it when

  • Task horizon > 10-20 tool calls
  • Deliverable is a document, codebase, or analysis
  • Multiple sub-topics benefit from isolated context
  • You want a research / coding agent architecture

Skip it when

  • Short tool-use loops
  • Latency-critical single-turn answers
  • You have a task shape that doesn't fit the default prompt
  • You want fine-grained control — build on LangGraph primitives directly
