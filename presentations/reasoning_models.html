PARADIGM

The Shift to Variable Test-Time Compute

Fixed Compute (Standard Models)

  • One forward pass per token, regardless of difficulty
  • "What is 2+2?" gets the same compute as "Prove Fermat's Last Theorem"
  • Fundamentally mismatched to difficulty distribution
  • Capability fixed at training time

Variable Compute (Reasoning Models)

  • Generate a variable-length reasoning chain before the answer
  • Easy problems: 0–50 reasoning tokens
  • Hard problems: 5,000–100,000+ reasoning tokens
  • Capability increases with inference-time budget

Two Scaling Axes

Training compute: larger models, more data → determines base capability. Test-time compute: more reasoning tokens → determines how effectively the model applies that capability. The interaction is multiplicative: a more capable base model benefits more from additional reasoning.

MECHANICS

How Chain-of-Thought Reasoning Works

Reasoning Chain Structure

  • Problem analysis: restate, identify variables and constraints
  • Strategy selection: compare approaches, pick the best
  • Step-by-step execution: show intermediate work
  • Self-verification: re-check calculations, test edge cases
  • Error correction: backtrack and try different approach if needed

Example

Find f'(2) where f(x)=2x³-3x²+x-5

Step 1: f'(x) = 6x²-6x+1
  (power rule for each term)

Step 2: f'(2) = 6(4)-12+1
  = 24-12+1 = 13

Verify: 6×4=24 ✓
  24-12=12 ✓  12+1=13 ✓

CoT Prompting vs Reasoning Models

CoT prompting ("think step by step") is prompt-dependent, produces unverified reasoning, and allocates uniform effort. Reasoning models are trained to reason, optimised for reasoning quality via RL, and allocate effort proportional to difficulty.

MODELS

OpenAI o1 / o3 / o4-mini

o1 (Sep 2024)

  • First commercial reasoning model
  • Hidden reasoning trace (user sees summary only)
  • 5–60s latency per response
  • Dramatic improvement on math, code, science
  • Weaker on non-reasoning tasks

o3 (Early 2025)

  • Better quality, reduced latency
  • Tool use during reasoning: search, code exec, browse integrated into chain
  • Configurable effort: low / medium / high
  • Explicit quality-latency-cost control

o4-mini (Apr 2025)

  • Reasoning in a smaller, cheaper model
  • Makes reasoning accessible for cost-sensitive apps
  • Less deep reasoning but far more affordable
  • Strong on code and structured tasks

MODELS

Claude Extended Thinking and DeepSeek-R1

Claude Extended Thinking

  • Thinking traces are visible to developers (thinking blocks in API)
  • Configurable budget (1K–128K thinking tokens)
  • Streaming: watch thinking in real-time
  • Tool use during thinking: search, code exec integrated into reasoning
  • Transparency enables debugging and trust

DeepSeek-R1

  • Open-weight reasoning model (MIT license)
  • Trained with RL on chain-of-thought (GRPO algorithm)
  • Key insight: reasoning emerges from RL alone without SFT on reasoning traces
  • Distilled into smaller models (1.5B–70B)
  • Comparable to o1 on math and code benchmarks

TRAINING

How Reasoning Training Works

Process Reward Models (PRMs)

  • Score each reasoning step, not just the final answer
  • Trained on human annotations of step-correctness
  • Enable credit assignment: which step caused failure?
  • Guide tree search: prune unpromising branches early
  • Better than outcome rewards for complex multi-step problems

Outcome Reward Models (ORMs)

  • Score only the final answer (correct / incorrect)
  • Simpler to train (no step-level annotation needed)
  • Model must discover good reasoning strategies via exploration
  • DeepSeek-R1 used outcome rewards + GRPO
  • Key finding: reasoning emerges from RL without explicit CoT training

Training Pipeline

Stage 1: Pre-train a base model. Stage 2: RL with reasoning rewards — the model learns to generate useful reasoning chains by trial and error. Stage 3: Rejection sampling — generate many reasoning chains, keep the ones that produce correct answers, use them for further SFT. Stage 4: Distillation — train smaller models on the reasoning traces of the larger model.

SCALING

Test-Time Compute Scaling Laws

The Scaling Curve

  • Accuracy improves log-linearly with reasoning tokens on many benchmarks
  • 2× more thinking tokens ≈ constant accuracy improvement
  • Diminishing returns at extremes (easy problems plateau fast, impossible problems never solve)
  • Sweet spot depends on task difficulty × model capability

When Test-Time Compute Helps

  • Helps: math, coding, logic, multi-step planning, verification-heavy tasks
  • Helps: problems with well-defined correctness criteria
  • Doesn't help: factual recall (model knows it or doesn't)
  • Doesn't help: creative writing (no single correct answer)
  • Doesn't help: tasks where the model lacks fundamental knowledge

TRADEOFFS

Latency, Cost, and Faithfulness

Latency

  • Reasoning tokens are generated before the answer
  • Simple tasks: 5–10s
  • Complex tasks: 30–120s+
  • Not suitable for real-time / low-latency apps
  • Streaming thinking mitigates perceived wait

Cost

  • Reasoning tokens are billed (output token pricing)
  • 10K reasoning tokens on o3 ≈ 10× cost of standard response
  • Budget controls are essential for production
  • Small reasoning models (o4-mini) reduce cost 5–10×

Faithfulness

  • Does the model actually use its reasoning chain?
  • Evidence is mixed: models sometimes reach correct answers via wrong reasoning
  • "Unfaithful" reasoning may be post-hoc rationalisation
  • Hidden reasoning (o1) makes verification impossible

PRACTICE

Distillation and Deployment

Distilling Reasoning

  • Train smaller model on reasoning traces from larger model
  • DeepSeek-R1 distilled to 1.5B–70B models
  • Distilled models reason faster and cheaper
  • Quality loss: 5–15% on benchmarks vs teacher
  • Sweet spot: 7B–32B distilled reasoners for production

Deployment Patterns

  • Routing: send easy tasks to standard model, hard tasks to reasoning model
  • Budget tiers: low/medium/high effort per user tier or task type
  • Verification: use reasoning model to verify standard model's output
  • Hybrid: standard model with optional "think harder" button

Limitations

Overthinking: model wastes tokens on problems it could solve quickly. Hallucinated reasoning: confident but wrong reasoning steps. Circular reasoning: repeating the same argument without progress. Sycophantic reasoning: reasoning toward what the user wants to hear rather than the truth.

SUMMARY

Key Takeaways

The Paradigm Shift

  • AI capability now scales on two axes: training compute and test-time compute
  • Reasoning models generate variable-length thinking chains before answering
  • Accuracy improves log-linearly with reasoning tokens
  • RL on chain-of-thought is the key training technique
  • Reasoning can emerge from RL alone (DeepSeek-R1)

Practical Guidance

  • Use reasoning models for math, code, logic, planning
  • Don't use them for simple factual recall or creative tasks
  • Set budget controls — reasoning tokens are expensive
  • Consider routing: standard model for easy, reasoning for hard
  • Distilled models (7B–32B) offer good quality/cost balance
  • Extended thinking with visible traces aids debugging
