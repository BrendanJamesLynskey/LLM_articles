The Attention Bottleneck

Self-attention computes an N×N score matrix — quadratic in sequence length. For a 128K-token sequence, the attention matrix alone requires ~30 GB in FP16.

O(N²) Memory

Score matrix S and probability matrix P must be materialized in HBM for standard attention.

O(N²) IO

S and P are written then read back from HBM — the dominant cost at long sequences.

FlashAttention Fix

Never materialize the N×N matrix. Tile to SRAM, fuse ops, achieve O(N) memory.

GPU Memory Hierarchy

Key Insight

SRAM is ~10x faster than HBM but ~4000x smaller. Any algorithm that repeatedly accesses HBM is leaving performance on the table.

Memory-Bound

Attention has high FLOPs but also high data movement — GPU arithmetic units are faster than HBM can feed them.

FlashAttention: Three Core Principles

Tiling

  • Divide Q, K, V into blocks that fit in SRAM
  • Compute attention one tile at a time
  • S and P never materialized in HBM
  • Only final output O written back

Online Softmax

  • Incrementally update max & sum as new key blocks arrive
  • Rescale prior results with exp(m_old - m_new)
  • Mathematically exact — no approximation
  • Only O(N) auxiliary storage

Kernel Fusion

  • Matmul + softmax + mask + dropout fused into one kernel
  • Eliminates HBM round-trips between ops
  • Avoids kernel launch overhead
  • Maximizes arithmetic intensity

The Tiling Algorithm

for each query block Q_i (outer loop):
  O_i = 0, m_i = -inf, l_i = 0

  for each K_j, V_j block (inner loop):
    Load Q_i, K_j, V_j from HBM -> SRAM
    S_ij = Q_i * K_j^T        # B_r x B_c tile
    m_ij = rowmax(S_ij)
    P_ij = exp(S_ij - m_ij)
    l_ij = rowsum(P_ij)

    m_new = max(m_i, m_ij)
    l_new = l_i*exp(m_i - m_new)
           + l_ij*exp(m_ij - m_new)

    O_i = O_i * (l_i*exp(m_i-m_new)/l_new)
        + P_ij*V_j * (exp(m_ij-m_new)/l_new)
    m_i, l_i = m_new, l_new

  Write O_i to HBM

Exact Result

After all K-V blocks, O_i is numerically identical to standard attention. No approximation.

Block Sizes

B_r, B_c chosen so working set fits in SRAM. Typical values: 64–128 on A100.

IO Complexity

Θ(N²d² / M) — proven optimal for exact attention under streaming model.

Online Softmax Trick

Why It Works

The factor exp(m_old - m_new) rescales all prior contributions when a new block yields a larger max. After all blocks, the result is identical to full-row softmax.

Storage

Only O(N) auxiliary storage: one max and one sum per query row, versus O(N²) for full attention matrix.

FlashAttention • 05 — Online Softmax
06

Backward Pass & Memory Savings

Recomputation Strategy

  • Forward: store only O and softmax stats (m, l)
  • Discard S and P entirely
  • Backward: recompute each P tile on-the-fly from stored stats
  • ~25% more FLOPs but negligible wall-clock cost (memory-bound)

Standard Attention

  • N=32K → S & P: ~2 GB per head per layer
  • N=16K → OOM at batch ≥ 4

FlashAttention

  • N=32K → softmax stats: ~16 MB
  • N=16K → runs at batch 16
  • Over 100x memory reduction

FlashAttention-2: Better Partitioning

FA-1 reached only 25–35% of A100 peak matmul throughput. FA-2 (2023) closed the gap:

Fewer Non-Matmul FLOPs

Deferred final rescaling until after inner loop, reducing non-matmul ops by ~25%. Tensor cores sit idle during scalar ops.

Sequence Parallelism

FA-1 parallelized over batch & heads only. FA-2 adds parallelism over sequence length in the outer loop — critical for small-batch inference.

Warp-Level Partitioning

Different warps process different query portions against shared K, V. Eliminates redundant reads and improves occupancy.

Result

1.7–2.0x speedup over FA-1, reaching 50–73% of A100 theoretical matmul throughput. Native GQA/MQA support added.

FlashAttention-3: Hopper Optimizations

FA-3 (2024, NeurIPS Spotlight) targets H100 hardware-specific features:

TMA + Warp Specialization

Tensor Memory Accelerator handles async HBM↔SRAM transfers. Producer warps load data via TMA while consumer warps compute on tensor cores concurrently.

Pingpong Scheduling

Two warp groups alternate: one executes GEMMs while the other runs softmax, then swap. Hides softmax latency behind GEMM execution — overlapping both in time.

FP8 Tensor Cores

H100 FP8 runs at 2x throughput of BF16. Block-wise quantization with per-tile scaling factors mitigates accuracy loss.

Incoherent Processing

Random orthogonal transforms spread outliers across dimensions before FP8 quantization, reducing quantization error by 2.6x vs. naive FP8.

FlashAttention • 08 — FA-3
09

FA-3: Pingpong GEMM-Softmax Pipeline

Why Pingpong?

Softmax (exp, reductions, rescaling) cannot use tensor cores. Without pingpong, tensor cores idle during softmax phases.

The Interleave

WG-A does GEMM while WG-B does softmax, then they swap roles. TMA prefetches next tile asynchronously throughout.

Result

Up to 840 TFLOPS BF16 (85% utilization) and 1.2+ PFLOPS FP8 on H100.

FlashAttention-4: The Blackwell Challenge

Announced by Tri Dao at Hot Chips 2025. Blackwell shifts the hardware balance:

New Bottleneck

Tensor core throughput scaled dramatically, but SFUs (for exp()) and shared memory bandwidth did not keep pace. Softmax is no longer easily hidden.

Solution Approach

Algorithm-kernel co-design specifically targeting Blackwell. Four key innovations address the asymmetry.

FlashAttention • 10 — FA-4 Challenge
11

FA-4: Key Innovations

Conditional Softmax Rescaling

  • Only rescale when new row max is sufficiently larger
  • ~90% of rescaling operations skipped
  • Dedicated correction warp group handles remainder
  • If no thread needs rescaling, entire warp skips
  • Result still numerically identical

Software-Emulated Exponential

  • Hardware MUFU.EX2 has limited throughput on Blackwell
  • Distribute exp() between HW unit and FMA emulation
  • Leverages idle FMA units during softmax
  • Effectively doubles exp() throughput

CuTeDSL Implementation

  • Python DSL from NVIDIA CUTLASS team
  • Replaces hand-written CUDA C++
  • Faster iteration and easier maintenance
  • Produces highly optimized code via CUTLASS

Multi-Stage Async Pipeline

  • MMA warps + softmax warps + correction warps
  • Accumulators live in TMEM (not registers)
  • Reduces register pressure
  • 2-CTA MMA mode for backward pass
FlashAttention • 11 — FA-4 Innovations
12

FA-4: Conditional Rescaling Deep Dive

The Condition

Only rescale when m_new >> m_old. In practice the row maximum converges quickly, making most rescaling redundant.

Warp-Level Skip

If no thread in a warp needs rescaling, the entire warp skips the operation. This is a per-warp predication, not per-thread.

Still Exact

The correction warp group ensures all deferred rescaling is applied. The final output is numerically identical to standard attention.

Key Takeaways

Core Principle

Optimize data movement, not arithmetic. Tile to SRAM, fuse operations, never materialize the N×N matrix in HBM.

Exact, Not Approximate

Online softmax with incremental rescaling yields numerically identical results to standard attention — across all four generations.

Four Generations

FA-1: Tiling → FA-2: Parallelism → FA-3: Hopper pingpong + FP8 → FA-4: Blackwell conditional softmax + CuTeDSL

Hardware Co-Design

Each version targets the specific bottleneck of its generation: IO (Ampere), pipeline stalls (Hopper), SFU throughput (Blackwell).
