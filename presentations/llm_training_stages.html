LLM Training Stages
OVERVIEW

The Three-Stage Pipeline

Modern LLM development follows a multi-stage pipeline, each stage with distinct objectives, data requirements, and computational profiles.

Each stage builds on the previous, with feedback flowing backward to inform the next generation.

LLM Training Stages — 1
PRE-TRAINING

Objective & Formulation

The dominant objective is causal language modeling (next-token prediction): the model learns to predict the probability distribution over the next token given the preceding sequence.

  • Training loss = cross-entropy between predicted distribution and actual next token
  • Decoder-only architecture has become the universal paradigm (GPT-4, Claude, Llama, Gemini)
  • Encoder-decoder models (T5, UL2) use masked span prediction but are declining in usage

Why Next-Token Prediction Works

This deceptively simple objective, applied at sufficient scale, produces models with broad linguistic knowledge, reasoning ability, and world knowledge. Its simplicity, natural fit for generation, and favorable scaling properties drove the convergence on decoder-only architectures.

LLM Training Stages — 2
PRE-TRAINING

Data Collection & Curation

Modern LLMs train on 1T to 15T+ tokens. Raw web data is extremely noisy and must pass through a rigorous pipeline.

Data Sources

  • Web crawls (Common Crawl)
  • Digitized books & academic papers
  • Code repositories (GitHub, GitLab)
  • Forums & Q&A sites
  • Curated references (Wikipedia)

Curation Pipeline

  • Language ID — fasttext classifiers
  • Quality filtering — perplexity + classifier-based
  • Deduplication — exact, MinHash/LSH, line-level
  • Content filtering — toxic/PII removal

Llama 3 Data Mix

~50% web • ~25% code • ~15% books & papers • ~10% curated — only ~15% of raw Common Crawl survives filtering

LLM Training Stages — 3
PRE-TRAINING

Architecture Decisions

Model Size & Shape

  • 1B params (edge) to 1T+ params (frontier MoE)
  • Hidden dim, layers, heads, FFN dim
  • Deeper = better sequential reasoning
  • Wider = more capacity per layer

Positional Encoding

  • RoPE is the modern standard
  • Replaces learned absolute embeddings
  • Encodes relative position via rotation matrices
  • Enables extrapolation to longer contexts

Normalization

  • RMSNorm replaced LayerNorm
  • Pre-normalization is now standard
  • Improves training stability

Activations

  • SwiGLU replaced ReLU/GELU
  • Gated activations improve quality at same compute
  • Context: 4K-8K tokens at pre-training time

Scaling Laws

Loss decreases as a power law in model size (N), dataset size (D), and compute (C).

FrameworkKey FindingImplication
Kaplan (2020)Larger models are more sample-efficientPrioritize model size
Chinchilla (2022)Optimal ratio: D ≈ 20NBalanced scaling
Post-ChinchillaInference cost dominatesOvertrain smaller models

Llama 3 8B Example

Trained on 15T tokens — ~250 tokens/param, roughly 12x the Chinchilla-optimal ratio. Smaller model, cheaper to serve, quality approaches larger Chinchilla-optimal models.

LLM Training Stages — 5
PRE-TRAINING

Training Infrastructure

Cluster Scale

  • Llama 3 405B: 16,384 H100 GPUs
  • 8 GPUs/node via NVLink
  • InfiniBand or custom interconnect
  • Runs for 2-4 months

Parallelism Strategies

  • Tensor — split layers within node
  • Pipeline — layers across nodes
  • Data — replicate across groups
  • Expert — for MoE models (4D)

Mixed Precision

  • BF16 for fwd/bwd passes
  • FP32 master weights in optimizer
  • FP32 for loss & gradient accum
  • Halves memory, doubles throughput
LLM Training Stages — 6
PRE-TRAINING

Training Dynamics & Compute Cost

Hyperparameters

  • Learning rate: warmup → cosine decay to ~10% of peak
  • Batch size ramp: 4M → 16M tokens (Llama 3)
  • Frontier runs: 4-60M tokens/step
  • Loss curve: rapid drop → slow power-law decrease

Frontier Cost

  • Llama 3 405B: 3.8 × 1025 FLOPs
  • ~30.8M GPU-hours on H100s
  • $60-90M in compute alone
  • GPT-4: estimated >$100M

Common Failure Modes

  • Loss spikes — data quality, numerical issues; may require checkpoint rollback
  • Training divergence — excessive LR causes NaN loss
  • Data contamination — benchmark data in training set inflates metrics
  • Slow degradation — subtle issues, detectable only via scaling law comparison

Llama 3 Reliability

466 job interruptions during its 54-day training run — hardware failures, loss spikes, and other issues.

LLM Training Stages — 8
MID-TRAINING

What Is Mid-Training?

Continued pre-training after the initial run but before post-training alignment. Uses next-token prediction with modified data, context length, or architecture.

  • Tens of billions to low trillions of additional tokens
  • Term not widely used before 2024, but practice dates earlier
  • Now recognized as a deliberate, planned pipeline stage

Key Modifications

  • Data mixture shifts
  • Context length extension
  • Domain specialization
  • Architecture surgery (experimental)
LLM Training Stages — 9
MID-TRAINING

Long-Context Extension

RoPE Frequency Scaling

Increase base frequency (e.g., 10K → 500K-1M) to compress rotational frequencies, enabling position representation beyond original range.

  • NTK-aware interpolation — standard approach
  • YaRN — scales different frequency components independently
  • Preserves short-range resolution while extending long-range

Progressive Extension

Context extended in stages, not a single jump:

Llama 3 Context Ladder

8K → 16K → 32K → 65K → 128K
~800B tokens total mid-training

Data Requirements

Must use data with genuine long-range dependencies: book-length texts, long code files, multi-document sets, synthetic needle-in-a-haystack tasks.

LLM Training Stages — 10
MID-TRAINING

Domain Adaptation & Annealing

Domain Adaptation

  • Continue pre-training on domain-specific data
  • Demonstrated for code, math, medicine, legal, science
  • Key risk: catastrophic forgetting
  • Solution: mix domain data with 20-50% general data
  • Lower LR than peak pre-training (near final pre-training LR)

Data Annealing

  • Shift mixture toward higher-quality sources in later stages
  • Early training: broad, diverse data for pattern learning
  • Later training: curated data for refined gradient signal
  • Cosine LR decay to near zero during annealing
  • Llama 3: annealing over final 40M tokens

Code Llama

Llama 2 + 500B code tokens → context extended to 100K

DeepSeek-V2/V3

Multi-stage mid-training planned from the start of pipeline

Depth Upscaling

Experimental: duplicate layers (e.g., 32 → 48) to add capacity

LLM Training Stages — 11
POST-TRAINING

Purpose & Supervised Fine-Tuning

Post-training transforms a capable but unrefined base model into a useful, safe, and steerable assistant.

Supervised Fine-Tuning (SFT)

  • First post-training step
  • Curated instruction-response pairs
  • Next-token prediction on response only
  • 10K to millions of examples
  • Quality >> Quantity (LIMA: 1K examples can work)
  • Covers Q&A, code, math, creative writing, multi-turn chat

What Post-Training Teaches

  • Follow instructions
  • Produce helpful, coherent responses
  • Refuse harmful requests
  • Use tools & function calling
  • Multi-turn conversation

Compute Budget

1021–1023 FLOPs vs. pre-training's 1025. But the ratio is trending upward as post-training becomes a key differentiator.

LLM Training Stages — 12
POST-TRAINING

RLHF: Reward Modeling & PPO

PPO Complexity

  • 4 models simultaneously: policy, reference SFT, reward model, value model
  • Sensitive to KL penalty, clipping ratio, learning rate
  • Strongest optimization when reward model is accurate

Reward Hacking

Without KL constraint, the policy exploits spurious patterns in the reward model. The KL penalty keeps the policy close to the SFT starting point, preventing degenerate outputs.

LLM Training Stages — 13
POST-TRAINING

DPO & Preference Optimization Variants

Direct Preference Optimization eliminates the reward model and RL step entirely. It derives a closed-form mapping between rewards and optimal policies.

  • Increases log-prob of preferred responses
  • Decreases log-prob of dispreferred responses
  • Implicit KL constraint via temperature parameter
  • Requires only the SFT model + preference dataset
MethodKey Innovation
DPODirect closed-form optimization
IPORobust regularization vs. overfitting
KTOUnpaired examples (no comparisons)
ORPOCombines SFT + preference in one stage
SimPONo reference model needed

PPO vs. DPO at the frontier remains actively debated — PPO may be stronger with high-quality reward models; DPO is more stable and accessible.

LLM Training Stages — 14
POST-TRAINING

Safety, Constitutional AI & Tool Use

Constitutional AI

  • Principles-based alignment (Anthropic)
  • Model self-critiques against a constitution
  • Generates own preference data
  • RLAIF: AI generates preference judgments at scale
  • Comparable quality, dramatically lower cost

Safety Training

  • Red-teaming: human + automated adversarial testing
  • Safety-specific SFT on refusal examples
  • Safety RLHF with dedicated reward models
  • Multi-objective: helpfulness vs. harmlessness
  • Over-refusal problem: benign vs. harmful distinction

Tool Use & Chat Format

  • SFT on tool invocation examples
  • Function calling with proper schemas
  • Multi-step tool planning
  • Chat templates (ChatML, Llama format)
  • System prompt adherence
  • Multi-turn context management

LLM Training Stages — 16
CONCLUSION

Key Takeaways

Pre-Training

  • Foundation of all capabilities
  • Data curation is paramount
  • Scaling laws guide decisions
  • Massive infrastructure required
  • Overtrain for inference efficiency

Mid-Training

  • Now a deliberate pipeline stage
  • Context extension (8K → 128K+)
  • Domain specialization
  • Data quality annealing
  • Architecture modification (emerging)

Post-Training

  • SFT → RLHF/DPO → Safety
  • Primary differentiator between models
  • Iterative data flywheel
  • Growing compute investment
  • Least transparent stage

Future model development will look less like a single massive run and more like a carefully orchestrated multi-stage pipeline where each phase is optimized for its specific contribution.
