CONTEXT

From Server to Rack

Blackwell’s headline is not the chip — it is the rack. The GB200 NVL72 fuses 36 Grace CPUs and 72 Blackwell GPUs into one liquid-cooled cabinet that NVIDIA calls “one giant GPU.”

Compute

  • 72 Blackwell GPUs
  • ~1.4 EFLOPS FP4
  • 720 PFLOPS FP8
  • 208B transistors / GPU

Memory

  • 13.5 TB HBM3e
  • 17 TB Grace LPDDR5X
  • 30 TB unified coherent
  • 192 GB per GPU

Interconnect

  • NVLink 5 @ 1.8 TB/s/GPU
  • 130 TB/s bisection
  • 72-GPU NVLink domain
  • Liquid cooled, 120 kW rack

CHIP

The Blackwell GPU

Dual-Die Package

  • Two reticle-limit dies on TSMC 4NP
  • 10 TB/s NV-HBI die-to-die link
  • Presented as single logical GPU
  • 208B transistors total — 2.6× H100
  • 8 stacks HBM3e → 192 GB, 8 TB/s

2nd-Gen Transformer Engine

  • Native FP4 tensor cores
  • 2× throughput vs FP8
  • Per-block scaling and selective promotion
  • FP4 weights, BF16/FP8 attention common pattern

Decompression Engine

  • Hardware decompression of GPTQ/AWQ/NVFP4
  • Streamed decode from HBM → tensor cores
  • 2:4 structured sparsity acceleration

Throughput Highlights

  • ~20 PFLOPS FP8 dense
  • ~40 PFLOPS FP4 dense
  • 1,200 W per GPU TDP
  • Liquid cooling required

SUPERCHIP

The GB200 Superchip

Grace + 2× Blackwell

  • 72-core ARM Neoverse V2 CPU
  • 480 GB LPDDR5X — coherent with GPU memory
  • 900 GB/s NVLink-C2C between CPU and GPUs
  • Single shared address space
  • ~2,700 W per module total

Coherent Memory Implications

  • 1.5T-parameter model fits in CPU+GPU memory of one module
  • Transparent KV-cache spillover to LPDDR5X
  • Kernels deref CPU memory directly — no DMA bookkeeping

Power & Cooling

  • ~2.7 kW per module
  • Direct-to-chip cold plates
  • Coolant: 25°C in / 35°C out
  • Air cooling not feasible

Module Density

  • 2 GB200 modules per compute tray
  • 18 trays per NVL72 rack
  • 120 kW per rack peak

The Big Idea

NVL72 raises the NVLink scale-up domain from 8 GPUs to 72 GPUs. Tensor parallelism at TP=72 stays on full-speed NVLink — no InfiniBand cliff.

PROGRAMMING

One Giant GPU

What “One GPU” Means

  • 72 GPUs presented as a uniform TP group
  • Any pair of GPUs at full NVLink BW
  • NCCL all-reduce ~90% efficient at TP=72
  • Megatron-Core auto-tunes for NVL72 topology

Memory Aggregation

  • 1.8T-param model in FP8 fits in 1.8 TB HBM
  • Leaves 11+ TB for KV cache & activations
  • Multi-million-token contexts feasible
  • Thousands of concurrent sessions per rack

Disaggregated Inference

  • Prefill pool + decode pool in same NVLink domain
  • KV transfer over NVLink, not InfiniBand
  • Cost of cache shuttling now negligible

Pipeline Bubble Elimination

  • TP=72 obsoletes pipeline parallelism
  • ~25% better hardware utilization on 1T models
  • Bubble overhead drops to near-zero

INFERENCE

30× Faster Inference Claim

NVIDIA quotes ~30× inference throughput improvement vs H100 cluster of equivalent GPU count for trillion-parameter models. Where does that come from?

Source of speedupMultiplierMechanism
FP4 vs FP8~2.5×2nd-gen Transformer Engine
Tensor core throughput~2×Larger SM, faster mma
NVLink-only TP~6×No InfiniBand cliff at TP>8
Combined~30×Best case
Real-world typical8–15×Workload dependent

Where the Win Is Largest

Models too large for an 8-GPU NVLink domain. Below ~100B params, an H100 node is competitive. Above ~500B, NVL72 has no real peer.

TRAINING

Training at Rack Scale

3D Parallelism Collapse

  • Old: TP within node, PP across nodes, DP across cluster
  • New: TP×PP up to 72 within one rack
  • Pipeline bubble overhead drops to near-zero
  • ~25% better hardware utilization

Cross-Rack Scaling

  • 800 GB/s top-of-rack uplink
  • InfiniBand or Spectrum-X between racks
  • Megatron-Core auto-detects topology

Hierarchy

  • 130 TB/s NVLink within rack
  • ~few TB/s InfiniBand between racks
  • Slower Ethernet between datacenters

Practical Topologies

  • Hyperscalers building 16-rack and 32-rack pods
  • Foundation models trained in single pod
  • Multi-pod for 10T+ parameter targets

FACILITIES

Power, Cooling, Datacenter

Power Density

  • 120 kW/rack — 10× standard datacenter density
  • Most existing facilities cannot host without retrofit
  • Hyperscalers building purpose-built facilities
  • MS Wisconsin, Meta Louisiana, xAI Memphis Colossus

Cooling Stack

  • Direct-to-chip cold plates
  • Rear-door heat exchangers or in-rack CDUs
  • Supply ~25–30°C, return ~45°C
  • Closed-loop with secondary chilled-water loop

Cluster Scaling

1,000-GPU H100 cluster — 40 air racks @ 30 kW — 1.2 MW. 1,000-GPU NVL72 cluster — 14 racks @ 120 kW — 1.7 MW. The Blackwell cluster delivers ~5× FP8 throughput at 1.4× the power.
