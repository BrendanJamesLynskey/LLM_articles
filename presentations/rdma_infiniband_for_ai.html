CONTEXT

Why RDMA Matters for AI

Frontier training jobs run on tens of thousands of GPUs. A single gradient sync moves ~20 GB across the fabric every few hundred milliseconds. Network performance determines scaling efficiency — not GPU FLOPs.

Conventional TCP/IP

  • Kernel processes every packet
  • 10–50 µs round-trip
  • CPU-heavy at high BW
  • Drops on congestion — retry storms

RDMA

  • NIC writes directly to remote memory
  • 1–2 µs round-trip
  • Zero CPU involvement on data path
  • Lossless fabric — no drops by design

VERBS

RDMA Programming Model

Verbs

  • SEND/RECV — two-sided messaging
  • WRITE — one-sided remote write
  • READ — one-sided remote read
  • ATOMIC — CAS, fetch-and-add on remote memory

Queue Pairs

  • Send queue + receive queue per connection
  • Hundreds of QPs per node typical
  • NCCL pipelines work across many QPs

Memory Registration

  • Application registers memory regions with NIC
  • NIC reads/writes directly via DMA
  • No copies, no kernel involvement

Completion Queues

  • Hardware queue for completion events
  • Polling or interrupt-driven consumption
  • Zero-copy data delivery

FABRIC

InfiniBand: The Lossless Fabric

Lossless by Design

  • Credit-based flow control
  • Sender stops if receiver has no credits
  • No drops → no retransmission needed
  • Predictable tail latency

Centrally Managed

  • Subnet Manager (SM) computes routes
  • Pushes routing tables to all switches
  • Single deterministic path per pair
  • Sub-second SM failover

Topology

  • Fat-tree (Clos network) dominant
  • 1:1 or 2:1 oversubscription typical
  • Dragonfly for 10k+ GPU clusters

Fabric Health

  • Bit-error-rate degradation common
  • UFM monitoring critical
  • Misconfigured PFC → head-of-line blocking

RoCE

RDMA over Converged Ethernet

Why RoCE

  • Use standard Ethernet switches
  • No vendor lock-in
  • Integrate with broader datacenter fabric
  • Lower hardware cost

The Lossy Problem

  • Ethernet drops under congestion natively
  • Need PFC + ECN to fake lossless
  • Misconfig → deadlocks, head-of-line blocking
  • Operationally trickier than IB

Spectrum-X

  • NVIDIA’s AI-optimized Ethernet platform
  • Adaptive routing & packet spraying
  • Telemetry-driven congestion control
  • SuperNIC complements switches

Adoption

  • Hyperscalers moving RoCE for new clusters
  • Neoclouds tend to stay IB
  • Roughly equivalent perf when tuned

NCCL

NCCL on RDMA Fabrics

What NCCL Does

  • Topology discovery at startup
  • Opens hundreds of QPs per pair
  • Selects ring/tree algorithms per collective
  • Pipelines transfers across QPs
  • Hierarchical: NVLink intra-node, IB inter-node

Tunables

  • Channel count
  • Protocol selection (Simple/LL/LL128)
  • Buffer sizes
  • Adaptive routing on/off

GPUDirect RDMA

  • NIC reads/writes GPU memory directly
  • Bypass CPU memory entirely
  • Latency 5 µs → 1.5 µs
  • BW efficiency 60% → 90%
  • Requires GPU+NIC same PCIe root

The Scale Gap

NVLink 5 within an NVL72 rack provides 1.8 TB/s per GPU — ~18× the per-GPU bandwidth of XDR IB. This is exactly why intra-rack stays NVLink and only cross-rack uses RDMA.

OPS

Operational Realities

Failure Modes

  • Degraded links pass traffic but with elevated BER
  • Manifest as mysterious slowdowns
  • PFC misconfig → head-of-line blocking
  • SM failover causes brief routing inconsistency

Monitoring

  • NVIDIA UFM for IB
  • Per-link error counters in Prometheus
  • Alerting on rising error rates
  • BER, congestion, retransmits as KPIs

Topology-Aware Scheduling

  • Place job GPUs close in fabric topology
  • SLURM, Volcano, Kueue all aware
  • Same leaf switch or 1 spine hop
  • Avoids contention with other jobs

Cost

  • 1k-GPU IB fabric: $5–10M
  • RoCE 30–50% cheaper for equivalent BW
  • Hyperscalers move to RoCE; neoclouds stay IB

FUTURE

Optical & the Next Decade

Co-Packaged Optics

  • Optical engines on switch ASIC package
  • Broadcom, NVIDIA, Marvell all developing
  • Targets 1.6 Tbps and beyond
  • First deployments 2026–2027

Silicon Photonics

  • Ayar Labs, Lightmatter, Lightelligence
  • Replace electrical NVLink with optical equivalents
  • Tens of meters at lower power than copper
  • Could blur rack boundary entirely

Long-Term Direction

The boundary between scale-up (NVLink) and scale-out (IB/RoCE) is dissolving. NVLink-NW already extends the coherent domain across racks. Optical interconnects will push it further, eventually allowing datacenter-aisle coherent compute fabrics.

SUMMARY

The Network Is the Computer

What Matters

  • RDMA bypasses kernel for µs-scale latency & CPU-free transfers
  • InfiniBand is the mature lossless fabric; RoCE catches up via PFC/ECN tuning
  • NCCL hides everything but exposes tunables for the brave
  • GPUDirect RDMA is non-negotiable for serious AI clusters
  • Network failures are the leading cause of training job slowdowns at scale

Practitioner’s Lesson

The network is rarely the bottleneck when it works well, but when it doesn’t, no amount of GPU horsepower compensates. Investment in fabric monitoring, topology-aware scheduling, and NCCL tuning pays dividends that often dwarf algorithmic gains.
