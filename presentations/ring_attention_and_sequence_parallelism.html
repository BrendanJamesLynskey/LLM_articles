The Activation Memory Wall

As context windows expand to millions of tokens, even memory-efficient kernels cannot fit activations on a single GPU.

  • Attention output per layer: (batch, seq_len, hidden_dim)
  • Llama 3 class (d=8192), 128K tokens, BF16: ~2 GB per layer
  • Across 80 layers: ~160 GB of activations alone
  • KV cache for inference similarly massive
  • No single GPU can hold this

Why Sequence Parallelism?

Existing Parallelism Falls Short

  • Tensor parallelism splits weight matrices but not per-token activation memory
  • Pipeline parallelism distributes layers but each device still needs the full sequence
  • Data parallelism requires each device to process the complete sequence

Sequence Parallelism Fills the Gap

  • Splits the sequence itself across devices
  • Each device holds only 1/N of the tokens
  • Directly reduces activation memory per device
  • Orthogonal to all other parallelism dimensions
  • Also called context parallelism

Megatron-LM's early "sequence parallelism" distributed LayerNorm/dropout but not the attention computation itself

Ring Attention: Core Design

N devices arranged in a logical ring, each holding 1/N of the sequence.

  • Each device computes Q, K, V projections for its local chunk
  • Attention proceeds over N rounds
  • Each round: compute attention between local Q and current KV block
  • Between rounds: send KV block to next device, receive from previous
  • After N rounds: every Q has attended to all KV pairs

Key insight: Communication overlaps with FlashAttention computation, hiding latency entirely.

Overlapping Communication with Compute

While device i computes FlashAttention on the current KV block, it simultaneously sends that block to device i+1 and receives the next block from i-1.

Online Softmax Across Blocks

Inherited from FlashAttention

  • Each local FlashAttention produces a partial attention output and log-sum-exp statistics
  • As new KV blocks arrive, the device updates running statistics and rescales accumulated output
  • After all N rounds, the result is numerically identical to single-device attention

Why This Matters

  • No approximation whatsoever
  • Bit-for-bit equivalent to standard attention
  • Quality is exactly preserved

Round k on Device i

O_local, LSE_local = FlashAttn(Q_i, KV_k)

// Combine with running state
LSE_new = logaddexp(LSE_run, LSE_local)
scale_old = exp(LSE_run - LSE_new)
scale_new = exp(LSE_local - LSE_new)

O_run = scale_old * O_run + scale_new * O_local
LSE_run = LSE_new

Causal Masking & Load Imbalance

The Problem

With contiguous chunks and causal masking:

  • Device holding last chunk computes attention against all preceding tokens
  • Device holding first chunk only attends to itself
  • Up to 50% compute wasted from load imbalance

Striped Attention (Solution)

Interleaved token assignment via zigzag pattern:

  • Device 0 gets tokens: 0, 2N-1, 2N, 4N-1, ...
  • Each device's workload is approximately equal
  • Compatible with RoPE: tokens keep original position indices

DeepSpeed Ulysses

All-to-All Head Partitioning

A fundamentally different approach to distributing attention:

  • Partition Q, K, V along the head dimension, not sequence
  • All-to-all redistribute: each device gets full sequence for a subset of heads
  • Standard FlashAttention per device (no distributed softmax)
  • Another all-to-all restores original partitioning

Advantages

  • All-to-all collectives are highly optimised on modern interconnects
  • Higher bandwidth utilisation than point-to-point ring transfers
  • No online softmax overhead

Ulysses: The Head Count Limit

Fundamental Constraint

Max parallelism degree cannot exceed the number of KV heads

  • 32 query heads with GQA using 8 KV heads
  • Ulysses limited to 8 devices max
  • Combined with tensor parallelism (which also partitions heads), even fewer devices available

Ring Attention: No Head Limit

Partitions along sequence dimension, not heads

  • Scales with device count, not head count
  • Can use hundreds or thousands of devices
  • Only requires enough tokens per device for compute/comm overlap

Hybrid: Unified Sequence Parallelism

Best of Both Worlds

USP combines Ring Attention and Ulysses to match the network hierarchy:

  • Intra-node (NVLink, high BW): Ulysses partitions across heads for maximum efficiency
  • Inter-node (InfiniBand, lower BW): Ring Attention distributes along sequence, overlaps comm with compute

Why It Works

  • Ulysses is faster where bandwidth is abundant
  • Ring hides latency where bandwidth is scarce
  • Two-level hierarchy matches GPU cluster topology

Context Parallelism in Practice

Meta / Llama 3

  • Modified all-gather ring algorithm (pass-KV)
  • Concurrently all-gathers KV shards while computing attention
  • Used for 405B model training and inference

PyTorch

  • context_parallel API replaces standard SDPA with Ring Attention
  • Supports all-gather and all-to-all rotation methods
  • Drop-in replacement

NVIDIA

  • Megatron-core and TransformerEngine
  • Context parallelism as first-class parallelism dimension
  • Integrated with TP, PP, DP

All major frameworks now support sequence parallelism alongside data, tensor, and pipeline parallelism

Performance at Scale

Training: ringX on Frontier

  • Llama 3 8B with 1M token sequence length
  • 4,096 GPUs on Frontier supercomputer
  • 3.4x speedup over naive ring attention
  • 38% MFU — among highest reported for long-context training at HPC scale

Inference: Llama 3 405B

  • Pass-KV ring attention significantly outperforms tensor parallelism for multi-node inference
  • Advantage grows with number of nodes
  • TP bottlenecked by all-reduce latency across nodes
  • Ring attention overlaps comm with compute

Key Takeaway

Context parallelism becomes increasingly advantageous over tensor parallelism as the deployment scales beyond a single node, because ring communication overlaps with computation while all-reduce does not.

Implementation Details

RoPE Position Indices

  • Tokens must keep original position indices regardless of physical device placement
  • Striped/zigzag assignment: position = original index, not local index
  • Frequency tensors (freq_cis) must be sharded consistently

Padding & Variable Length

  • Sequence length must be evenly divisible by parallelism degree
  • Pass-Q variant for inference: sends queries (not KV) around ring
  • More efficient when new tokens are small relative to cached context

Composing Parallelism

  • DP + TP + PP + CP can all be composed
  • Communication patterns must be coordinated to avoid deadlocks
  • Process group meshes (Megatron-LM, TorchTitan) manage complexity

Pass-KV vs Pass-Q

  • Pass-KV: rotate KV blocks (standard Ring Attention)
  • Pass-Q: rotate query blocks instead
  • Pass-Q better for prefill with small query relative to long cached context

Key Takeaways

Ring Attention

  • Distributes attention across a ring of GPUs
  • Overlaps KV transfer with FlashAttention compute
  • Numerically identical to single-device attention
  • Scales linearly with device count, no head limit
  • Striped assignment solves causal load imbalance

DeepSpeed Ulysses

  • Partitions across attention heads via all-to-all
  • Faster for moderate parallelism (high BW utilisation)
  • Limited to number of KV heads
  • Standard FlashAttention kernel, no modifications

Hybrid Approach (USP)

Ulysses intra-node + Ring inter-node = best of both worlds, matching parallelism strategy to network bandwidth hierarchy. As models push toward million-token contexts, these techniques are essential infrastructure for making such contexts computationally feasible.
