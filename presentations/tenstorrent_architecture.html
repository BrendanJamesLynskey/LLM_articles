Grayskull

Proof-of-concept. 120 Tensix cores at 1 GHz. Validated the packet-based dataflow model for transformer inference.

Wormhole

Key innovation: native Ethernet links on every die edge. 80 cores with higher per-core perf. Direct chip-to-chip via PCB traces.

Blackhole

PCIe Gen5, GDDR6/HBM, wider matrix engines. Targets both inference and training. Competitive for 7B–70B models.

Conditional Execution Model

Data-Dependent Compute

  • Tensix cores inspect data before dispatching work
  • All-zero tiles after ReLU? Skip the matmul entirely
  • Sparse attention masks? Skip masked positions
  • Control RISC-V core checks packet metadata
  • Decides whether to dispatch to matrix engine

GPU Contrast

GPUs execute the same instruction across all warp threads regardless of data values. Zero tiles get the same compute as dense tiles. Conditional execution is a fundamental architectural advantage for sparse workloads.

Packet-Based Dataflow

  • Data moves as packets with metadata
  • Each packet: data + destination + dimensions + processing requirements
  • NoC routes dynamically based on runtime conditions
  • Computation triggered by data arrival
  • Natural pipelining across the mesh

Key Benefit

Reduces need for global synchronization barriers. Each Tensix core fires when its input data arrives, enabling fine-grained pipeline parallelism across the 2D mesh.

Ethernet Everywhere

Same Protocol at Every Level

  • On-chip: Tensix ↔ Tensix via Ethernet NoC
  • Chip-to-chip: die-edge Ethernet ports
  • Rack-to-rack: standard Ethernet links
  • No NVLink, no InfiniBand, no custom ASICs

Why This Matters

  • Single routing logic for all scales
  • No proprietary interconnect premium
  • Standard PCB traces for chip-to-chip
  • Familiar protocol for network engineers

Galaxy Multi-Chip Systems

Architecture

  • Multiple Wormhole/Blackhole chips tiled into 2D mesh
  • Each chip connected to neighbors via Ethernet
  • Scales to hundreds of chips
  • Flat, uniform network topology
  • No "near" vs "far" protocol distinction

Use Cases

  • Large-model inference: distribute weights across chips
  • Training: distribute data and gradients
  • Software stack treats entire mesh as single logical fabric

Scaling Advantage

No proprietary interconnect hardware needed. Standard Ethernet from die edge to rack scale. Compare: NVIDIA requires NVLink + NVSwitch + InfiniBand for equivalent multi-chip systems at $$$$ premium.

Software Stack

TT-Metalium

Low-level SDK. Direct control over Tensix cores, NoC, memory. Like writing CUDA kernels but for Tensix. Open-source on GitHub.

TT-NN

Optimized matmul, attention, convolutions, normalization. Handles tiling across cores and mesh parallelism. Supersedes TT-BUDA.

PyTorch 2.0 Integration

Custom backend for torch.compile(). Run standard PyTorch models with minimal code changes. Compilation + dispatch to Tensix hardware.

Current Limitations

Early Ecosystem

  • Software stack less mature than CUDA
  • Limited profiling tools and debuggers
  • Fewer community-contributed libraries
  • Many models need manual Metalium optimization
  • PyTorch frontend not yet fully optimized

Limited Model Zoo

  • Fewer models run optimally out-of-the-box
  • Peak performance often requires custom kernels
  • NVIDIA has thousands of optimized models
  • Tenstorrent model coverage growing but narrow

Dense Workload Gap

  • GPUs excel on dense, regular workloads
  • Massive HBM3 bandwidth + optimized kernels
  • Conditional execution advantage only with actual sparsity
  • Fully dense models see less benefit

Chicken-and-Egg Problem

  • Developers wait for hardware adoption
  • Customers wait for software maturity
  • Competing against NVIDIA's entrenched ecosystem
  • Open-source strategy aims to break the cycle
