CONTEXT

The Sequential Bottleneck

Every modern LLM is autoregressive: produce token N, then N+1, then N+2. This is fundamentally sequential — you cannot parallelize the inner loop.

Autoregressive Reality

  • 1,000 tokens of output = 1,000 forward passes
  • Decode bottleneck is latency, not FLOPs
  • Speculative decoding gives ~2–3× speedup at most
  • No path to dramatic improvement

The Diffusion Promise

  • Generate by parallel denoising
  • K steps to produce N tokens, K << N
  • Wall-clock latency depends on parallelism, not output length
  • Success in image generation: DALL-E, Stable Diffusion, Flux

CHALLENGE

Why Text Diffusion Is Harder

Pixels vs Tokens

  • Pixels are continuous values
  • Tokens are discrete
  • Gaussian noise has no obvious meaning on text
  • Score matching requires smooth distributions

Two Approaches

  • Continuous: embed tokens, diffuse in embedding space
  • Discrete: mask/transition tokens directly
  • Continuous lossy at the snap-back step
  • Discrete won the practical battle

DISCRETE

Discrete Diffusion: How It Works

Forward (Corruption)

  • Start with clean tokens
  • At each timestep, mask some tokens with [MASK]
  • Or transition them to other tokens stochastically
  • After T steps, sequence is fully masked

Reverse (Denoising)

  • Model learns to predict original tokens from partially-masked input
  • Related to BERT-style MLM training
  • Masking ratio varies across examples

Sampling Procedure

  • Start with fully-masked sequence of length N
  • Run K denoising steps (K ≈ 30)
  • At each step: predict masked positions, unmask some
  • K << N — that’s where the speed comes from

Unmask Order

  • Left-to-right (mimics autoregression)
  • Confidence-based (commit easy tokens first)
  • Random
  • Confidence-based usually best quality

TRADE

Speed vs Quality

The Quality Gap

  • Diffusion LMs 1–5% behind AR on standard benchmarks
  • Small but consistent
  • AR has cleaner probability factorization
  • AR commits one token at a time, less risk of errors

The Speed Win

  • 1,000-token output: 30 steps vs 1,000 steps
  • Each diffusion forward pass is ~1.5–2× an AR pass
  • Net: 5–10× faster wall-clock
  • Most pronounced for low-batch latency-sensitive workloads

The Sweet Spot

Diffusion wins when latency matters more than the absolute quality ceiling. Voice agents, code completion, real-time agents, long-form generation under deadline. AR still wins when batch throughput is king and quality is paramount.

HYBRID

Hybrid Approaches

AR-Diffusion

  • AR backbone produces draft
  • Diffusion refines the draft
  • Best of both: AR coherence + diffusion editability

Block Diffusion

  • Generate text in blocks of 8–16 tokens
  • Diffusion within block
  • Autoregression between blocks
  • Balance parallelism with coherence

Speculative Diffusion

  • Small diffusion drafter
  • Large AR verifier
  • Like speculative decoding but with diffusion drafts

USE CASES

Where Diffusion May Win

Long-Form Generation

  • 10k-token doc in 1–2s vs 10s for AR
  • Hard latency targets unmet by AR
  • Streaming less critical than completion

Code Editing

  • Long-range structure, easy for diffusion
  • Refine existing code naturally
  • Edit operations awkward for AR

Constrained Generation

  • Hard constraints (regex, templates) easier to enforce
  • Denoising can be conditioned throughout
  • AR struggles to backtrack

Real-Time Voice

  • 200 ms response vs 2s feels qualitatively different
  • Reduced sequential bottleneck
  • Inception Labs targeting this market

LIMITS

Open Problems

No KV Cache Equivalent

  • AR benefits hugely from KV caching
  • Diffusion has no obvious analog
  • Each step processes full sequence
  • Limited reuse between steps

Variable Length

  • AR stops on EOS naturally
  • Diffusion must commit to length up front
  • Workarounds exist but feel awkward

Streaming

  • AR streams naturally
  • Diffusion produces all-at-once
  • Streaming variants sacrifice parallelism win

Quality at Scale

  • Most diffusion LMs are 1–10B params
  • Behavior at 70B+ unknown
  • Whether gap with AR shrinks or grows is open

SUMMARY

An Alternative, Not a Replacement

What’s True

  • Diffusion LMs are real, working, in production (Mercury)
  • Not yet matching AR on quality benchmarks
  • Dramatically faster for latency-sensitive workloads
  • Architecture diversification is healthy for the field

Practitioner’s Lesson

Keep diffusion in the toolbox for applications where latency dominates. Mercury is good enough to use today for many tasks. The unusual performance profile — much faster but slightly lower quality — gives it a niche even if it never displaces autoregression.
