CONTEXT

Why GPU Interconnect Matters

A 70B model in FP16 occupies 140 GB — more than any single GPU. Multi-GPU execution requires constant synchronization. The system’s performance is bounded by GPU-to-GPU bandwidth and latency, not just per-GPU FLOPs.

Scale-Up Domain

  • GPUs that talk to each other at NVLink speeds
  • Microsecond latency, TB/s bandwidth
  • Coherent memory addressing
  • Tensor parallelism is efficient inside this domain

Scale-Out Domain

  • GPUs across nodes — PCIe + InfiniBand
  • 1-2 orders of magnitude lower bandwidth
  • Higher latency, no memory coherence
  • Only data parallelism / pipeline parallelism scale efficiently

SWITCH

NVSwitch Architecture

The Crossbar

  • Non-blocking switch for GPU-to-GPU traffic
  • Like an Ethernet switch but for NVLink packets
  • Single-hop routing within a switch group
  • Multi-hop across switch trays in larger topologies

NVSwitch 4 (DGX H100)

  • 64 NVLink 4 ports
  • 6.4 TB/s aggregate
  • 4 chips give full bisection on 8 GPUs

NVSwitch 5 (NVL72)

  • 72 NVLink 5 ports
  • Supports 576-GPU address space
  • 9 switch trays per NVL72 rack
  • SHARP in-network reduction

SHARP

  • Switch-side reduction for all-reduce
  • 1.5–2× effective collective BW
  • Originally an InfiniBand feature
  • Used automatically by NCCL

DOMAIN

Scale-Up Domain Growth

Generations of Domain Size

  • DGX-1 (P100): 8 GPUs, non-uniform mesh
  • DGX-2 (V100): 16 GPUs, uniform NVSwitch
  • DGX A100/H100: 8 GPUs in switch
  • NVL72 (Blackwell): 72 GPUs in switch
  • NVL576 (Rubin): 576 GPUs (planned)

Why It Matters

  • TP=72 obsoletes pipeline parallelism for trillion-param models
  • MoE all-to-all can stay on NVLink
  • ZeRO-3 / FSDP at full bandwidth
  • Reduced complexity in 3D parallelism config

The 8→72 Jump

  • Most important single transition since 2018
  • Makes models infeasible on prior gen practical now
  • Backbone of NVL72 commercial pitch

COLLECTIVES

Collective Communication Patterns

All-Reduce

  • Dominant collective in DL
  • Ring (BW-optimal) vs Tree (latency-optimal)
  • NCCL chooses based on size
  • SHARP gives in-network reduction
  • 80–95% of theoretical peak

All-Gather / Reduce-Scatter

  • The two halves of all-reduce
  • Used by ZeRO-3 and FSDP
  • NVLink-only on rack
  • InfiniBand brutal on these

All-to-All

  • Dominant in MoE routing
  • Stresses bisection BW
  • ~80% efficiency on NVL72
  • 30–50% on InfiniBand clusters

Why MoE Loves NVLink

All-to-all bandwidth on NVL72 is roughly 10× per-GPU what an H100+IB cluster offers. MoE training and serving are the workloads with the largest NVLink benefit.

NCCL

NCCL: The Communication Library

What NCCL Does

  • Topology-aware collective API
  • Detects NVLink, NVSwitch, PCIe, IB at startup
  • Selects ring/tree/double-binary algorithms per call
  • Used by PyTorch, JAX, Megatron, DeepSpeed, vLLM
  • Supports compute-comm overlap via CUDA streams

Hierarchical Algorithms

  • Intra-node first over NVLink
  • Inter-node second over IB
  • Reduces volume crossing slow fabric
  • Critical for >1k-GPU jobs

Tunables

  • NCCL_NCHANNELS
  • NCCL_PROTO (Simple/LL/LL128)
  • NCCL_BUFFSIZE
  • NCCL_TOPO_FILE
  • Production teams sweep these

FUTURE

Beyond the Rack: NVLink Network

NVLink-NW

  • NVLink over fiber between racks
  • ~1 µs cross-rack latency
  • Enables NVL576 designs
  • First version connects 2× NVL72

The Convergence

  • Spectrum-X Ethernet shares many goals
  • RoCE handles RDMA-style traffic
  • Boundary scale-up vs scale-out is blurring
  • Datacenter as one coherent fabric eventually

Competition

  • AMD Infinity Fabric / XGMI
  • Intel Xe Link with integrated RDMA Ethernet
  • UALink consortium — open scale-up standard
  • 1024-accelerator domain target

Operational Reality

  • NVLink failures cause subtle slowdowns
  • Monitoring via DCGM
  • Topology-aware scheduling required
  • Switch reboots evacuate 8–16 GPUs

SUMMARY

Why NVLink Defines the Era

Compute Growth

  • ~10× per generation
  • FLOPs aplenty
  • Limited by data movement

NVLink Growth

  • ~30× per-GPU bandwidth
  • ~9× domain size
  • SHARP-style in-network compute

Result

  • Tensor parallelism at scale
  • MoE all-to-all efficient
  • Pipeline parallelism becomes optional

Key Insight

NVLink + NVSwitch are the unsung heroes of modern LLM training. Most parallelism strategies that “just work” today were impractical without them. Interconnect is the rate-limiter, and NVIDIA invested in it earlier than competitors.
