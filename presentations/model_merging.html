CONTEXT

The Surprising Observation

Take two fine-tunes of the same base model. Average their weights. The result is often as good as either parent, sometimes better — with no retraining and no labeled data.

Why This Should Not Work

  • Neural network weights are high-dimensional
  • Random points in weight space are noise
  • Averaging two random nets → garbage

Why It Actually Works

  • Fine-tunes share a base — nearby in weight space
  • Linear mode connectivity holds
  • Both fine-tunes live in the same low-loss basin
  • Averaging stays inside the basin

FRAMEWORK

Task Vectors

Definition

τtask = θfinetuned − θbase

The displacement that fine-tuning introduced. Captures everything learned during fine-tuning.

Operations

  • Negation: subtract to unlearn a task
  • Addition: combine multiple tasks
  • Scaling: dampen or amplify a task
  • Sign election: resolve conflicts

Most Methods Are Operations on Task Vectors

  • Naive averaging = uniform task vector sum
  • Task Arithmetic = weighted task vector sum
  • TIES = sparsified, sign-aligned sum
  • DARE = randomly sparsified sum
  • Most differences are how to handle conflicts

TIES

TIES-Merging

1. Trim

  • Keep top-k% of params by magnitude
  • Zero out the rest
  • Typically 20–40%
  • Discards noisy small changes

2. Elect Sign

  • For each param, vote on sign
  • Largest magnitude wins
  • Disagreeing vectors get zeroed at that param
  • The key conflict resolution step

3. Disjoint Merge

  • Average remaining sign-aligned values
  • Add to base model
  • Result: cleanly merged task vector

Why It Helps

Naive averaging cancels out conflicting weights (“0.5 + −0.3 = 0.1” loses both signals). TIES picks one sign per parameter and discards dissent. Conflicts are resolved, not averaged through.

DARE

DARE: Drop And Rescale

The Procedure

  • Randomly zero out fraction p of task vector params
  • Typical p = 0.7–0.99
  • Rescale survivors by 1/(1−p)
  • Like dropout for task vectors

The Surprise

  • Dropping 99% often has minimal impact
  • Fine-tuning info is heavily redundant
  • A sparse skeleton suffices for the task

DARE + TIES

  • Apply DARE first (sparsify)
  • Then TIES (sign-elect, merge)
  • Currently the dominant production recipe
  • Hundreds of merged models on Hugging Face use this

Why It Works

Fine-tuning produces highly correlated weight updates. Most can be removed without losing the underlying behavior — you just need a small representative subset.

TOOLING

mergekit

The dominant open-source tool for model merging. Created by Charles Goddard in 2023. Supports all major methods, plus exotic operations like layer-wise interpolation and Frankenmerging.

Features

  • Linear, Task Arithmetic, TIES, DARE, SLERP and more
  • YAML config interface
  • Output in standard HF format
  • Layer-wise weights (different per depth)
  • Frankenmerging (interleave layers from different models)

PRACTICE

When It Works

It Helps When

  • Constituent models share a base
  • Tasks are complementary, not conflicting
  • Fine-tunes are mild, not extreme
  • You have a way to evaluate the merge

It Doesn’t When

  • Models from different architectural lineages
  • Tasks in genuine conflict (safety vs helpfulness)
  • Heavy retraining moved models too far
  • Benchmarks don’t reflect actual use case

Open LLM Leaderboard Era

For most of 2024, the leaderboard was dominated by merges. Real improvements were genuine; some were benchmark overfitting. v2 leaderboard added more diverse evals and de-emphasized opaque merges. Merging is still common but no longer dominates.

THEORY

Practice Ahead of Theory

What We Know

  • The methods empirically work
  • Linear mode connectivity holds for fine-tunes
  • DARE-TIES with density 0.5 is a strong default
  • Same-base requirement is real

What We Don’t

  • Why mode connectivity holds
  • Predicting merge quality in advance
  • Whether better methods are still possible
  • Relationship to ensembling, distillation

SUMMARY

Why It Matters

The Value Proposition

  • Combine capabilities of multiple parents
  • No retraining, no labeled data, no GPU compute
  • Minutes of CPU work to produce a new model
  • Compositional — every Hugging Face fine-tune is a potential ingredient

The Bigger Picture

Model merging demonstrates how much “free” capability is sitting in the open-source ecosystem, waiting to be combined. Network effects: every quality fine-tune released adds value to every other one through possible merges. A genuine advantage of the open-source ecosystem over closed alternatives.
