OVERVIEW

An agent is a latency pipeline

The loop

  • LLM call → parse → tool invocation → observation → LLM call → …
  • Each step is a round-trip with its own latency budget.
  • Final answer is emitted after N such steps.

The sum

Tagent ≈ Σsteps ( TLLM + Ttool + Torch + Tmemory )

  • TLLM is well-studied and tracked.
  • Ttool, Torch, Tmemory are where most agent time actually lives.
agent-perf / 02
TAILS

Tail amplification is the core problem

Tails compound

  • Per-step p99 = 2 s. Agent trajectory = 8 steps.
  • P(any step ≥ p99) = 1 − 0.998 ≈ 7.7%.
  • About 1 run in 13 hits at least one p99 event.
  • End-to-end p99 ≠ 8 × per-step p99 — it lies between Σp50 and Σp99.

What to optimize

  • Tighten p99.9 on per-step latency, not p50.
  • Cutting median TPOT improves the happy path.
  • Cutting p99.9 TTFT is what makes agents feel reliable.
agent-perf / 03
ROUND-TRIPS

Round-trip count beats per-hop speed

Parallel tool calls

  • Model emits N independent tool calls; orchestrator fires them concurrently.
  • Supported by OpenAI parallel function calls and Anthropic parallel tool use.
  • Latency saving ≈ (N − 1) × Ttool.

Speculative tool execution

  • Start running the most-likely tool while the model is still emitting the call.
  • If correct, result ready the moment parsing completes.
  • Safe only for idempotent reads.
  • Streaming tool-call arguments unlock the same idea cleanly.

CPU

CPU coupling — the hidden budget

Where the ms go

  • Tokenization of a 4K prompt: 10–40 ms.
  • Prompt assembly (system + tools + history + memory).
  • Response parsing (state-machine JSON extraction).
  • Orchestration framework overhead: 5–30 ms per step.
  • Serialization across process boundaries.

Tactics

  • Cache tokenized prefixes — system prompt, tool schemas.
  • Pre-compile templates; re-render only what changes.
  • Stream-parse tool calls as soon as each completes.
  • Pin orchestrator to same host as inference endpoint when latency > scalability.

NETWORK

Network coupling — the speed of light floor

The physics

  • Intra-DC: 0.1–1 ms one-way.
  • Intra-region: 5–20 ms.
  • Inter-continent: 70–150 ms.
  • 10-step agent × 1 inter-regional call each = ~2 s on network alone.

Defenses

  • Co-locate agent with most-called tools and MCP servers.
  • Pool HTTP/2 / gRPC connections.
  • Hedged requests — collapse p99 of 2 draws toward p50 of 1.
  • Per-tool deadlines with "tool failed" fallback.
  • Fan out independent calls in parallel.
agent-perf / 06
STORAGE

Storage coupling — RAG, memory, KV cache

Vector DB / RAG

  • HNSW top-k: 1–5 ms local, 5–20 ms remote, up to 100 ms cold.
  • Query re-embedding: 20–60 ms GPU, 100s of ms CPU.
  • Cache embeddings; batch when possible.

Prompt / KV cache

  • Long static system prompt → huge TTFT win from caching.
  • Requires session pinning across load-balancer replicas.
  • KV offload to CPU / NVMe → decode latency 10–100× worse. Not for interactive agents.

Agent memory

  • Local SQLite: ~1 ms. Remote doc store + vector search: ~50 ms.
  • 12 steps × 30 ms memory read = 360 ms of pure memory latency per run.

Fixes

  • Read memory once per run when stable.
  • Pre-fetch memory in parallel with the LLM call.
  • Keep short-term context in-process.

METRIC

Task goodput — the honest headline

Definition

  • Successful tasks per unit time per dollar.
  • Or per unit GPU, per unit operator pain.
  • "Successful" means correct AND within deadline.

Why it's the right metric

  • Ties model quality + inference perf + tool quality + coupling costs into one number.
  • A brilliant but slow agent has low task goodput.
  • A fast but wrong agent has low task goodput too.
  • Aligns incentives across the whole stack.

End-to-end

  • E2E p50 ≈ 8 × 515 ms ≈ 4.1 s.
  • E2E p99 lies between Σp50 and Σp99 — simulate, don't add.

Where to spend effort

  • LLM dominates p50 → spec decoding / smaller model cuts E2E to ~2.5 s.
  • Tool p99 dominates the tail → co-locate or hedge for reliability.

PROCESS

Critical-path analysis

Per step, measure

  • TLLM — TTFT + decode
  • TCPU — orchestration, parsing, tokenization
  • Tnet — tool-call network
  • Tstorage — memory / RAG / vector reads
  • Twait — blocked on external event

Then rank and act

  • Optimize the dominant term first. Ignore the rest.
  • LangSmith / Langfuse / Arize Phoenix / OTel break this down automatically.
  • The critical path is rarely what you think before measuring.

CONCLUSION

Six rules for agent performance

Measurement

  • Measure per-step latency broken down by component — not just the model call.
  • Care about p99 and p99.9, not means. Agents roll the dice each step.
  • Report task goodput as the headline — successful tasks per deadline per dollar.

Architecture

  • Collapse round-trips: parallel tools, speculative execution, streaming arguments.
  • Co-locate with most-called tools and most-accessed storage; the speed of light is real.
  • Treat CPU orchestration as a first-class performance surface.

A fast inference engine is necessary but nowhere near sufficient for a fast agent. The coupling points around the model are where most of the latency — and most of the optimization opportunity — actually live.
