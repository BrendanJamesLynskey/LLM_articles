OVERVIEW

What LangChain Solves

Wrapper Problem

  • Every provider ships a different SDK
  • OpenAI, Anthropic, Gemini, Mistral, Bedrock, xAI, Ollama, vLLM ...
  • Rewriting adapters per team is waste
  • Unify under BaseChatModel

Composition Problem

  • Real apps are pipelines, not single calls
  • Retrieve → rerank → prompt → model → parse
  • LCEL expresses pipelines as Runnables
  • Pipe operator | binds them

Integration Problem

  • ~100s of model / vector-store / loader adapters
  • Pre-built so you don't write glue
  • Community packages move fast — pin versions
  • Swap an integration in one import

LCEL

The Runnable Protocol

Shape

class Runnable(Generic[Input, Output]):
    def invoke(self, input, config=None) -> Output: ...
    def stream(self, input, config=None) -> Iterator[Output]: ...
    def batch(self, inputs, config=None) -> list[Output]: ...
    async def ainvoke(...) -> Output: ...
    async def astream(...) -> AsyncIterator[Output]: ...
    async def abatch(...) -> list[Output]: ...

Who implements it

  • Chat models (BaseChatModel)
  • Prompts (ChatPromptTemplate)
  • Output parsers (StrOutputParser, JsonOutputParser)
  • Retrievers (BaseRetriever)
  • Tools (BaseTool)
  • Your own functions via RunnableLambda

LCEL

Composition with the Pipe Operator

Basic chain

from langchain_openai import ChatOpenAI
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.output_parsers import StrOutputParser

prompt = ChatPromptTemplate.from_messages([
    ("system", "You are a concise writer."),
    ("human", "Summarise: {paragraph}"),
])
model = ChatOpenAI(model="gpt-4o", temperature=0)

chain = prompt | model | StrOutputParser()
chain.invoke({"paragraph": "LangChain is ..."})

Parallel branches & lambdas

from langchain_core.runnables import RunnableParallel, RunnableLambda

analysis = RunnableParallel(
    summary   = summary_prompt   | model | StrOutputParser(),
    sentiment = sentiment_prompt | model | StrOutputParser(),
    keywords  = keywords_prompt  | model | StrOutputParser(),
)

pipe = RunnableLambda(fetch_docs) | analysis

MODELS

Chat Models, Tools, Structured Output

Tool binding

from langchain_core.tools import tool

@tool
def get_weather(city: str) -> str:
    """Weather for a city."""
    ...

agent_model = model.bind_tools([get_weather])
reply = agent_model.invoke("Weather in Porto?")
reply.tool_calls  # [{"name": "get_weather", ...}]

Structured output

from pydantic import BaseModel

class Person(BaseModel):
    name: str
    age: int

extractor = model.with_structured_output(Person)
extractor.invoke("Alice is 32.")
# -> Person(name="Alice", age=32)

Provider-native JSON modes where available; retry parser as fallback.

RAG

A Minimal RAG Chain

from langchain_chroma import Chroma
from langchain_openai import OpenAIEmbeddings
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.runnables import RunnablePassthrough
from langchain_core.output_parsers import StrOutputParser

retriever = Chroma(
    collection_name="docs",
    embedding_function=OpenAIEmbeddings(),
).as_retriever(search_kwargs={"k": 8})

def format_docs(docs):
    return "\n\n".join(d.page_content for d in docs)

prompt = ChatPromptTemplate.from_template("""
Answer from the context. If unsure, say so.
<context>{context}</context>
Question: {question}
""")

rag = (
    {"context": retriever | format_docs, "question": RunnablePassthrough()}
    | prompt | model | StrOutputParser()
)
rag.invoke("What is PagedAttention?")

Seven lines, production-shaped. Swap the retriever, swap the model — one import each.

PRODUCTION

Reliability and Observability

Retries & fallbacks

primary = ChatAnthropic(model="claude-sonnet-4-5")
backup  = ChatOpenAI(model="gpt-4o")

model = (
    primary
    .with_retry(stop_after_attempt=3)
    .with_fallbacks([backup])
)

One-line transformation of an existing Runnable.

Tracing

export LANGSMITH_TRACING=true
export LANGSMITH_API_KEY=lsv2_...
export LANGSMITH_PROJECT=my-app

Every chain run appears in LangSmith with token counts, latency, and full step inputs/outputs. Attach tags & metadata to every invoke for post-hoc filtering.

TAKEAWAYS

Production Playbook

Do

  • Build around langchain-core + dedicated provider packages
  • Use LCEL (|) for every pipeline
  • Use .with_structured_output() for structured data
  • Use LangGraph for any loop
  • Trace with LangSmith from day one
  • Tag & metadata every invoke
  • Pin integration versions

Don't

  • Start with legacy chain classes
  • Hide prompts inside helpers you can't edit
  • Build agent loops in LCEL (use LangGraph)
  • Import from langchain-community when a dedicated package exists
  • Skip tracing until "later"
