Etched: Company Overview

The Company

  • Founded 2022
  • Backed by Y Combinator
  • Raised $120M+ in venture funding
  • Chip: Sohu
  • Mission: fastest possible transformer inference

The Thesis

If transformers remain the dominant AI architecture, a chip that hardcodes transformer operations into silicon will deliver unmatched inference efficiency.

If transformers are displaced, the chip is worthless.

The Bet

Etched is not building a general AI accelerator. Sohu is a single-architecture ASIC — it can run transformers and nothing else. This is the most extreme specialization bet in the AI hardware space.

The Specialization Spectrum

GPU Approach

General-purpose CUDA cores, programmable pipelines, caches. Can run any workload but carries overhead.

AI ASIC (TPU, Groq)

Hardcode matrix multiply + general NN ops. Efficient for many architectures, not just transformers.

Sohu: Extreme ASIC

Hardcode softmax, layer norm, attention, transformer-specific matmuls. Maximum efficiency, zero flexibility.

Eliminating GPU Overhead

H100: Where Transistors Go

  • Instruction fetch & decode pipeline
  • Warp schedulers (per SM)
  • Register files for arbitrary programs
  • L1 / L2 cache hierarchies
  • Memory management units
  • Branch predictors
  • PCIe / NVLink interface logic

<50% of transistors are Tensor Cores doing actual matmul

Sohu: Reclaimed Silicon

  • No general-purpose CUDA cores
  • No programmable shader pipelines
  • No instruction decoders for non-transformer ops
  • No hardware branch predictors
  • No complex cache hierarchy
  • Dataflow predetermined by architecture

Freed silicon → more compute, wider data paths, larger SRAM

Result

Etched claims roughly an order of magnitude more useful compute per watt and per mm² of silicon compared to a GPU running the same transformer workload.

Sohu Chip Design

Fabrication

  • Manufactured at TSMC
  • Advanced process node
  • Competitive transistor density vs H100 (TSMC 4N)
  • Same foundry as NVIDIA, Apple, AMD

On-Chip SRAM Strategy

  • Aggressive SRAM allocation for weights + activations
  • High bandwidth, low latency (sub-ns)
  • No HBM dependency on critical path
  • Trade: less capacity for more bandwidth

Memory Trade-Off

PropertySRAMHBM
BandwidthVery highModerate
LatencySub-ns~100s ns
DensityLowHigh
Power/bitLowHigher

Design Implication

Limited per-chip SRAM capacity requires multi-chip configurations for large models. Weights distributed via tensor parallelism across chips.

Eight-Chip Server Configuration

Reference Deployment

  • 8 Sohu ASICs per server
  • High-bandwidth, low-latency inter-chip links
  • Tensor-parallel model partitioning
  • Each chip holds a weight slice in local SRAM
  • Collective SRAM holds full Llama 70B

Design Balance

8 chips balances per-chip SRAM limits against practical server form factors and interconnect complexity. Low inter-chip latency preserves throughput.

The Transformer Longevity Bet

Why Transformers Will Dominate

  • Cross-modal dominance: NLP, vision, speech, protein folding, robotics
  • Scaling laws: no saturation at current parameter counts
  • Ecosystem lock-in: tools, frameworks, training recipes, deployment infra
  • Investment inertia: billions spent on transformer-optimized hardware
  • Research momentum: vast majority of top papers use transformers

The Existential Risk

  • RNNs dominated until transformers displaced them overnight
  • Mamba / S4: state-space models with linear sequence scaling
  • RWKV: recurrence-based alternative at scale
  • Hybrid approaches: mixing attention with other mechanisms
  • If any alternative wins: Sohu has zero residual value

The Core Question

Sohu is a physical embodiment of a bet on one specific computational graph. It cannot be reprogrammed, reconfigured, or adapted. The economic calculation: cost savings from purpose-built hardware over 3–5 years vs. the risk of stranded assets if the field shifts.

vs. Groq

Groq's TSP is programmable — it can run CNNs, MoE routing, future architectures. Sohu trades this flexibility for higher per-op efficiency.

vs. Cerebras

WSE has 40 GB on-chip SRAM on a single die, supports training + inference, runs any architecture. More versatile but less specialized.

vs. GPUs

GPUs support every architecture ever devised. Organizations using Sohu still need GPUs for training, non-transformer inference, and experiments.

Risk Analysis

  • Cannot be reprogrammed or reconfigured
  • Zero residual value if transformers displaced
  • History: RNNs → transformers happened fast

Early Reception & Skepticism

Sources of Enthusiasm

  • Genuine GPU inference limitations are well-documented
  • Clean-sheet ASIC design is intellectually compelling
  • YC backing and $120M+ funding signal credibility
  • Transformer dominance shows no near-term signs of ending
  • Inference cost reduction is an urgent industry need

Sources of Skepticism

  • Pre-production hardware — no shipping product yet
  • Claims require superior silicon and optimized multi-chip data movement
  • Many AI hardware startups have failed to deliver
  • GPU software (TensorRT-LLM, vLLM) keeps improving
  • The ASIC graveyard is large

What Would Validate Sohu

Independent benchmarks on production hardware, at realistic batch sizes, with optimized GPU baselines (TensorRT-LLM + FP8 + continuous batching + speculative decoding). Until then, all performance claims remain provisional.

Key Takeaways

The Promise

  • 500K+ tok/s on Llama 70B per 8-chip server
  • 10x+ throughput vs H100 (claimed)
  • Order-of-magnitude improvement in inference economics

Bottom Line

Etched Sohu is the logical extreme of the specialization thesis. If you know exactly what computation you need, you can build hardware that does it with unmatched efficiency. The potential reward is enormous. The potential risk is equally stark. For ML engineers: watch closely, wait for production benchmarks, and maintain GPU flexibility.
