LANGGRAPH

LangGraph Server & Platform

LangGraph Platform (hosted)

  • Push a graph → managed HTTPS API
  • Managed Postgres checkpointer + store
  • Thread + run APIs, SSE streaming
  • HITL endpoints
  • LangSmith tracing pre-wired

LangGraph Server (self-host)

  • Same binary as the platform
  • Same REST + SSE API
  • Bring your own Postgres
  • Run on K8s / ECS / Cloud Run / VM
  • Built with langgraph build
# Local dev with Studio UI + hot reload
langgraph dev

# Build production Docker image
langgraph build -t my-agent

# Deploy (platform or self-host)

SERVER API

LangGraph Server Endpoints

EndpointPurpose
POST /threadsCreate a thread
GET /threads/{id}/stateCurrent thread state
POST /threads/{id}/runsStart a run (optionally streaming)
POST /threads/{id}/runs/streamSSE stream of events
POST /threads/{id}/runs/waitBlocking wait for completion
POST /threads/{id}/historyFull checkpoint history
POST /runs / /runs/streamStateless runs (no thread)
GET/POST /assistantsNamed, versioned graph configurations
GET/POST /store/itemsCross-thread (long-term) memory

Client SDKs (langgraph-sdk Python / @langchain/langgraph-sdk TS) wrap these endpoints. Only needed in client apps — graphs don't need them to call themselves.

CLI

LangGraph CLI Flow

langgraph.json manifest

{
  "dependencies": ["."],
  "graphs": {
    "agent": "./src/my_agent/graph.py:graph",
    "research": "./src/my_agent/research.py:research_graph"
  },
  "env": ".env"
}

Dev loop

  1. Write your graph in Python/TS
  2. Declare it in langgraph.json
  3. langgraph dev — local server + Studio
  4. Iterate with hot reload
  5. langgraph build -t my-agent — Docker
  6. Deploy: managed platform or your K8s

VERSIONING

Stability & Pinning

Core stability

  • langchain-core, langgraph: conservatively versioned, rare breaking changes
  • Integration packages: move fast — provider SDKs change underneath
  • Pin minors in prod; let core float within a minor

pyproject.toml example

[project]
dependencies = [
  "langchain-core>=0.3,<0.4",
  "langchain-anthropic>=0.3,<0.4",
  "langgraph>=0.2,<0.3",
  "langgraph-checkpoint-postgres>=2.0,<3.0",
  "langsmith>=0.2,<0.3",
]

TAKEAWAYS

Summary

Discipline

  1. Smallest set of packages that meets your needs — core + one provider + one vector store + LangSmith handles most apps
  2. Add LangGraph when the agent grows a loop
  3. Add deepagents when it grows a long horizon
  4. Add langgraph-sdk only in client apps that consume a deployed server
  5. Pin integration packages; let -core float within a minor

Python leads ecosystem breadth & presets. TS leads web/edge deployment. Choose the language your stack already lives in.
