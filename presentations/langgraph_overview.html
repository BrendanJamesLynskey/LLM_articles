WHY

Why a Graph

Agents are not chains

  • Chains are DAGs — fixed direction, no loops
  • Agents cycle: call model, run tool, feed result back, repeat
  • Branching, retries, approvals, sub-agents
  • A pipeline abstraction can't express any of these cleanly

What you get for free

  • Durable execution — resume from last checkpoint
  • Human-in-the-loop — pause, approve, resume
  • Time travel — branch from any checkpoint
  • Streaming — tokens & structured events
  • Parallel fan-out — Send API
  • Subgraphs — compose graphs

STATE

State and Reducers

Declaration

from typing import Annotated, TypedDict
from langgraph.graph import StateGraph, START, END
from langgraph.graph.message import add_messages

class AgentState(TypedDict):
    messages: Annotated[list, add_messages]
    scratchpad: dict
    budget_remaining_usd: float

Custom reducers

def merge_counters(old: dict, new: dict) -> dict:
    return {**old, **{k: old.get(k, 0) + v for k, v in new.items()}}

class State(TypedDict):
    counters: Annotated[dict, merge_counters]

Parallel nodes updating the same field go through the reducer — fan-out is safe.

BUILD

Minimal ReAct Agent

@tool
def get_weather(city: str) -> str:
    """Current weather for a city."""
    return f"{city}: 18°C, clear"

tools = [get_weather]
model = ChatOpenAI(model="gpt-4o").bind_tools(tools)
tools_by_name = {t.name: t for t in tools}

class AgentState(TypedDict):
    messages: Annotated[list, add_messages]

def call_model(state): return {"messages": [model.invoke(state["messages"])]}

def run_tools(state):
    out = []
    for call in state["messages"][-1].tool_calls:
        result = tools_by_name[call["name"]].invoke(call["args"])
        out.append(ToolMessage(content=str(result), tool_call_id=call["id"]))
    return {"messages": out}

def should_continue(state):
    return "run_tools" if state["messages"][-1].tool_calls else END

graph = StateGraph(AgentState)
graph.add_node("agent", call_model)
graph.add_node("run_tools", run_tools)
graph.add_edge(START, "agent")
graph.add_conditional_edges("agent", should_continue, ["run_tools", END])
graph.add_edge("run_tools", "agent")
app = graph.compile()

langgraph-overview / 06
HITL

Human-in-the-Loop

Static breakpoint

app = graph.compile(
    checkpointer=checkpointer,
    interrupt_before=["run_tools"],
)
# First invoke: runs up to run_tools, stops.
app.invoke(input, config)
# Approve & resume:
app.invoke(None, config)

Dynamic interrupt

from langgraph.types import interrupt, Command

def confirm(state):
    decision = interrupt({"action": state["proposed"]})
    return {"approved": decision == "yes"}

# Resume:
app.invoke(Command(resume="yes"), config)

State persists across the pause. Process crashes don't lose approvals-in-flight.

Send (fan-out)

from langgraph.types import Send

def fan_out(state):
    return [Send("research", {"query": q})
            for q in state["subquestions"]]

graph.add_conditional_edges("planner", fan_out, ["research"])

DEPLOY

Deployment Options & Conclusion

Three paths

  • Embedded — app.invoke inside FastAPI / Next.js / worker
  • LangGraph Platform — managed API, threads, HITL, streaming
  • Self-hosted LangGraph Server — same binary on your K8s

When to use LangGraph

  • Agent has a loop
  • Needs to pause for humans
  • Needs to survive restarts
  • Needs to branch
  • Needs multi-agent composition

Overkill for: single-turn extraction, plain Q&A RAG — LCEL is enough.
