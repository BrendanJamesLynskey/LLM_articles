OVERVIEW

Cerebras Systems

Founded in 2016 by Andrew Feldman and a team of semiconductor veterans, Cerebras challenged a core assumption: that wafers must be diced into small dies before use.

Their thesis: if the bottleneck in AI compute is data movement between chips, eliminate chip boundaries entirely. Build a single processor from an entire 300mm silicon wafer.

The result is a purpose-built AI accelerator that trades commodity flexibility for raw, single-chip performance.

Key Milestones

  • 2016 — Company founded
  • 2019 — WSE-1 unveiled (1.2T transistors)
  • 2021 — WSE-2 on 7nm (2.6T transistors)
  • 2024 — WSE-3 on 5nm (4T transistors)
  • 2025 — Cerebras Inference cloud GA

HARDWARE

WSE-3 Specifications

Compute

  • 4 trillion transistors
  • 900,000 AI-optimized cores
  • TSMC 5nm process
  • 46,225 mm² die area

Memory

  • 44 GB on-chip SRAM
  • 21 PB/s memory bandwidth
  • Each core has local SRAM
  • No off-chip HBM dependency

Interconnect

  • 2D mesh on-chip fabric
  • Per-core dedicated router
  • Dynamic fault rerouting
  • Single-hop neighbor communication

The WSE-3 is ~56x larger than the largest GPU die (~800 mm²) and provides ~5,000x more memory bandwidth than a single HBM3e-equipped GPU.

MEMORY

The Bandwidth Advantage

Why This Matters

LLM inference is memory-bound, not compute-bound. The bottleneck is how fast you can feed weights to ALUs. On-chip SRAM eliminates the off-chip memory wall entirely.

SRAM vs HBM Trade-off

SRAM: Massive bandwidth, limited capacity (44 GB). HBM: Lower bandwidth, higher capacity (80-192 GB). Cerebras solves the capacity gap with MemoryX.

cerebras / 06
SYSTEM

MemoryX & SwarmX

MemoryX holds model weights in DRAM/flash. SwarmX streams them to the WSE at sufficient bandwidth to saturate all 900K cores. Model size is decoupled from on-chip memory capacity.

cerebras / 07
ARCHITECTURE

Weight Streaming Execution

GPU: Weight-Stationary

  • Weights loaded into HBM, kept resident
  • Activations flow through the network
  • Model must fit in aggregate HBM
  • Multi-GPU sharding for large models
  • Inter-chip communication overhead

Cerebras: Weight-Streaming

  • Activations stay stationary on SRAM
  • Weights stream through from MemoryX
  • Model size decoupled from on-chip memory
  • No multi-chip sharding needed
  • Zero inter-chip communication

The same hardware runs 70B or 405B parameter models — only the MemoryX configuration and streaming schedule change.

TRAINING

Training Capabilities

The weight streaming model adapts to training: both forward-pass weights and backward-pass gradients stream through the compute fabric.

Cerebras has demonstrated training convergence on models up to hundreds of billions of parameters, with near-linear scaling across multiple CS-3 systems for data-parallel training.

However, the training ecosystem remains less mature than the inference story. Most large-scale training still occurs on GPU clusters due to established tooling and workflow integration.

Training Highlights

  • Data-parallel across multiple CS-3s
  • Pipeline parallelism for huge models
  • Near-linear scaling demonstrated
  • 100B+ parameter convergence shown

Caveats

  • GPU tooling more mature for training
  • Smaller community of practitioners
  • Custom debugging/profiling required

SOFTWARE

Software Stack

PyTorch Integration

Standard PyTorch code compiles to WSE via custom backend. GPT-style decoders, encoder-decoder, and MoE architectures well supported.

Compiler Role

Partitions computation across 900K cores, schedules weight streams from MemoryX, handles placement, routing, and buffer management on the 2D mesh.

SCALING

Multi-System & Cloud

Multi-CS-3 Configurations

For workloads exceeding single-system capacity, SwarmX interconnects multiple WSE-3 wafers. This enables:

  • Pipeline parallelism for extremely large models
  • Data parallelism for training scale-out
  • Shared MemoryX pools across systems

Near-linear scaling has been demonstrated for data-parallel training workloads across multiple CS-3 nodes.

Cerebras Inference Cloud

  • API access to WSE-backed models
  • No on-premises hardware needed
  • Fastest time-to-first-token available
  • Targets real-time / conversational AI
  • Generally available since 2025

Target Users

Developers and enterprises wanting latency benefits of wafer-scale compute without capital investment in CS-3 hardware.

LIMITS

Limitations & Considerations

Cost

Each CS-3 represents a significant capital investment. Difficult to justify for smaller orgs or diverse non-AI workloads. Per-system price far exceeds individual GPU servers.

Ecosystem Maturity

Software stack improving but lacks the breadth and community of CUDA. Not all PyTorch ops supported. Custom debugging and profiling tooling required.

Model Constraints

While MemoryX extends capacity, there are practical limits to streaming bandwidth. Some novel architectures may not map efficiently to the 2D mesh topology.

Deployment Requirements

  • Liquid cooling infrastructure mandatory
  • ~23 kW per system power requirement
  • Specialized datacenter planning

Competitive Landscape

  • GPU memory tech improving (HBM4 roadmap)
  • NVIDIA interconnects getting faster
  • Other AI ASICs competing (TPU, Trainium)

SUMMARY

Key Takeaways

1. Physics Over Economics

Cerebras bets that data movement physics matters more than commodity processor economics. The WSE-3 eliminates chip boundaries to remove bandwidth bottlenecks.

2. Inference Sweet Spot

For latency-sensitive LLM inference, wafer-scale compute delivers unmatched per-request performance. The weight streaming model decouples model size from on-chip memory.

3. Architectural Simplicity

No tensor parallelism, no inter-chip communication, no HBM bottleneck. A single-system solution for models that would require multi-node GPU clusters.

4. Open Questions

Ecosystem maturity, cost accessibility, and whether GPU memory/interconnect improvements will narrow the gap. Worth serious evaluation for latency-critical inference.
