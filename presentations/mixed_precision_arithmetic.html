The Core Idea

Not all operations in a transformer need the same numerical precision. Mixed precision assigns higher precision to sensitive operations and lower precision where it can be tolerated.

Low Precision (FP8/INT4/FP4)

  • Feed-forward GEMMs
  • Attention projection weights
  • KV cache storage

High Precision (BF16/FP32)

  • Softmax computation
  • Layer normalisation
  • Residual additions
  • Embedding & lm_head

Result: most compute benefits from low-precision acceleration while the critical path maintains full fidelity.

BF16: The De Facto Baseline

BF16 retains FP32's 8-bit exponent while truncating the mantissa to 7 bits. This preserves the full dynamic range while halving memory.

  • Models trained in BF16 can be served directly -- no conversion needed
  • Handles activation outliers gracefully due to wide range
  • All lower-precision strategies are measured against BF16 as baseline

FP8: The Production Workhorse

2x Memory Reduction

Half the footprint of BF16 with negligible accuracy loss (<0.5% perplexity change)

Hardware Accelerated

Native tensor core support on Hopper & Blackwell at same throughput as INT8

Minimal Friction

Simple post-training calibration step. In vLLM, just set the quantization parameter

Tensor cores compute in FP8 but accumulate in FP32, preventing rounding error compounding across thousands of MACs.

INT8 & INT4: Integer Formats

Integer formats distribute precision uniformly across the range, while floating-point formats concentrate it logarithmically near zero.

  • INT8 -- 256 levels, well-suited to pre-processed weight distributions
  • INT4 -- 16 levels, 4x memory reduction over BF16
  • Outliers consume disproportionate range in integer formats
  • FP8 often beats INT8 for activation quantization
  • INT4 with per-group scaling remains competitive for weight-only quantization

FP4 & Microscaling Formats

Plain FP4 (E2M1)

  • 2 exp + 1 mantissa
  • Only 8 magnitude levels
  • Impractical without scaling

MXFP4 (OCP Spec)

  • Block of 32 elements
  • E8M0 (power-of-2) scale
  • ~3.8x effective reduction

NVFP4 (Blackwell)

  • Block of 16 elements
  • E4M3 scale + FP32 per-tensor
  • ~3.5x reduction, near-FP8 accuracy

Microscaling Principle

A shared high-precision scale factor across a small block of low-precision elements. The block's max-magnitude element is recovered at near-FP8 precision, even though stored in only 4 bits. This two-level approach gives NVFP4 finer-grained adaptation to local value distributions.

Prefill vs. Decode Asymmetry

Prefill Phase

  • Processes entire prompt at once
  • Compute-bound -- large matrix multiplications
  • Benefits from low-precision tensor cores
  • Best with W4A4 or W8A8

Decode Phase

  • Generates one token at a time
  • Memory-bandwidth-bound -- reads full model weights
  • Benefit is reducing data volume from HBM
  • Best with W4A16 or W8A16

Phase-adaptive mixed precision: switch strategy between prefill and decode for optimal efficiency.

Layer-Wise Sensitivity

Not all layers respond equally to quantization:

  • First & last layers are most sensitive -- they establish internal representations and produce logit distributions
  • Middle layers tolerate aggressive quantization
  • Embeddings & lm_head are frequently kept at higher precision

Advanced Methods

  • SliM-LLM -- salience metrics per layer/group
  • Any-Precision LLM -- bitplane encoding for dynamic runtime precision

The Memory Bandwidth Wall

Modern GPUs have far more compute than memory bandwidth, especially at low precisions.

H100 SXM Example

  • 1,979 TFLOPS FP8 compute
  • 3.35 TB/s HBM bandwidth
  • 70B model at FP8 = ~70 GB weights
  • One decode step = read all 70 GB
  • 70 GB / 3.35 TB/s = ~21 ms/token
  • Max ~48 tokens/sec (single batch)

Strategy: W8A8 FP8

The Simplest Production Path

  • Weights quantized statically via post-training calibration
  • Per-tensor or per-channel scaling factors
  • Activations quantized dynamically per-token at runtime
  • Softmax, LayerNorm, residuals, lm_head stay in BF16/FP32

Transformer Engine

NVIDIA's automated FP8 management maintains a history of activation magnitudes and updates scaling factors to track distribution over time.

Strategy: W4A16 Weight-Only INT4

GPTQ

  • Post-training quantization using inverse Hessian
  • Per-group scaling (group size 128)
  • 1-3% perplexity increase typical
  • Broad ecosystem support

AWQ

  • Activation-aware weight quantization
  • Protects salient weight channels
  • 1-2% perplexity increase typical
  • Often slightly better than GPTQ

At inference: weights dequantized to BF16 on-the-fly before GEMM via optimised kernels (e.g., Marlin) that fuse dequant + matmul.

Best for: single/low-batch decode, consumer GPUs, cost-sensitive cloud serving -- where the bottleneck is reading weights, not computing.

KV Cache Quantization

The KV cache stores attention state for all processed tokens and can dominate GPU memory at long contexts or large batches.

  • BF16 -> FP8 KV cache halves memory
  • Enables 2-3x larger batch sizes or longer contexts
  • FP8 preferred over INT8 for KV -- better dynamic range for attention values
  • KIVI explores asymmetric 2-bit KV cache quantization

Outlier Handling Strategies

SmoothQuant

Migrates quantization difficulty from activations to weights via per-channel scaling. Mathematically equivalent transformation that produces smoother activation distributions.

QuaRot

Applies Hadamard rotations to weight & activation matrices, redistributing outlier energy across all channels for more uniform, quantization-friendly distributions.

Microscaling

Block structure provides inherent outlier resilience -- each block's scale factor adapts to the local maximum, preventing one outlier from affecting the entire tensor.

Activation outliers -- values several standard deviations from the mean -- appear in specific channels and can dominate the quantization range, compressing the majority of values into just a few levels.

Calibration & Scaling Granularity

Static vs. Dynamic

  • Static -- computed once during calibration, zero runtime cost, requires representative data
  • Dynamic -- computed per input at runtime, more robust to outliers, adds overhead
  • Weights always use static (they don't change)
  • Activations: dynamic per-token is more robust but slower

Granularity Spectrum

  • Per-tensor -- single scale, coarsest, fastest
  • Per-channel -- one scale per output channel
  • Per-group -- 32 or 128 elements, finest practical
  • Per-block (microscaling) -- optimised for tensor core layout (NVFP4 = 16 elements)

Trend: finer granularity as hardware adds native support.

Emerging Directions

Phase-Adaptive Precision

Progressive mixed precision during decode: start at higher precision, gradually lower it as sequence progresses. Early tokens are more influential than later ones.

Format-Specialised Quantization

MR-GPTQ tailors quantization to FP4 properties with fused Hadamard rotations. Achieves 3.6x layer-wise speedup on Blackwell B200.

Mixed Precision for MoE

Expert-aware allocation: frequently activated experts get higher precision, rarely used experts tolerate aggressive quantization based on activation frequency.

1-Bit & Sub-2-Bit

BitNet (1.58-bit ternary {-1,0,1}) eliminates multiplication entirely. BinaryMoS uses token-adaptive scaling. Promising for on-device power efficiency.
