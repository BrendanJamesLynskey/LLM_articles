Open Weights vs. Open Source

Open Weights

  • Trained weights released publicly
  • Sometimes includes inference code
  • Training data and pipeline not published
  • Most "open" LLMs fall here

Truly Open Source

  • Weights + training data + code + hyperparameters
  • Fully reproducible from scratch
  • Examples: OLMo (AI2), LLM360
  • Rare but growing

Mistral & Mixtral

Mistral 7B

  • Sliding window attention for efficiency
  • Matched or exceeded Llama 2 13B
  • Excellent for single-GPU deployment

Mixtral 8x7B

  • Sparse MoE: 47B total, only 13B active per forward pass
  • Near-70B quality at a fraction of compute
  • Landmark efficiency demonstration

Also released: Mistral Large, Codestral (code-focused), Mistral Small under various licenses.

Qwen (Alibaba Cloud)

  • Qwen 2.5 spans 0.5B to 72B parameters, with a popular 32B "sweet spot" variant
  • Specialized variants: Qwen-Coder (code) and Qwen-Math (mathematics)
  • Strong performance on both English and Chinese benchmarks
  • Released under Apache 2.0 — fully permissive

Attention Mechanisms That Matter

Grouped-Query Attention (GQA)

Shares KV heads across query heads, reducing KV-cache size. Used by Llama 3, Mistral, Qwen 2.5, Gemma 2.

Multi-head Latent Attention (MLA)

DeepSeek's innovation: projects keys and values into a lower-dimensional latent space. Maximum KV-cache compression.

Sliding Window Attention

Limits attention to a local window, reducing memory for long sequences. Trades long-range capability for efficiency. Mistral's approach.

Local-Global Interleaved

Gemma 2 27B alternates between local and global attention layers, reducing KV-cache while preserving long-range ability.

Practical Selection Criteria

Quantization Compatibility

Some models quantize more gracefully. Quantization-aware trained models perform best at INT4.

Community Tooling

Check for GGUF, GPTQ, AWQ, EXL2 on Hugging Face. Broader format support = easier deployment.

Context vs. KV Budget

Long context requirements compete directly with KV-cache memory. Match model context to your actual needs.

Fine-tuned Variants

A domain-tuned small model often outperforms a larger general-purpose one for your specific task.

Serving Frameworks

vLLM

  • De facto standard for production serving
  • PagedAttention + continuous batching
  • Tensor & pipeline parallelism
  • Supports most major architectures

llama.cpp

  • Ideal for consumer hardware
  • CPU/GPU splitting for partial offload
  • RPC backend for network-distributed inference
  • GGUF format ecosystem

TensorRT-LLM

  • Lowest latency on NVIDIA hardware
  • Requires model-specific compilation
  • Best for dedicated production deployments

exo

  • Peer-to-peer distributed inference
  • Runs across heterogeneous consumer devices
  • Pool together mismatched hardware

Key Takeaways

  • The 7-9B class (Llama 3.1 8B, Qwen 2.5 7B, Gemma 2 9B, Mistral 7B) provides the best quality-to-resource ratio at INT4/INT8
  • The 27-35B class is accessible on two consumer GPUs for higher quality needs
  • The 70B class remains the sweet spot for near-frontier quality on 4-8 small GPUs with 4-bit quantization
  • GQA and MLA are critical innovations for fitting models into limited VRAM
  • MoE architectures unlock larger effective models with less active compute
  • Each new release pushes the quality-per-parameter frontier forward
  • Check licenses before production deployment
